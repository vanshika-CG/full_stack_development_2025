# Basics of Algorithms and Mathematics

### What is an Algorithm and Pseudocode Writing

---

## 1. What is an Algorithm

An algorithm is a **finite, well-defined sequence of instructions** that takes some input, performs a series of computational steps, and produces an output to solve a given problem.

The word "algorithm" comes from the name of the Persian mathematician **Al-Khwarizmi** (9th century), whose work on arithmetic was later translated into Latin.

> **Formal Definition (Knuth):**
> An algorithm is a finite set of rules which gives a sequence of operations for solving a specific type of problem, having five important features: Finiteness, Definiteness, Input, Output, and Effectiveness.

---

## 2. Five Essential Properties of an Algorithm

Every valid algorithm must satisfy all five of the following properties:

---

### 2.1 Finiteness

An algorithm must always terminate after a **finite number of steps**.

A procedure that runs forever is not an algorithm. Every instruction must eventually lead to a stopping point.

**Example:** A loop that counts from 1 to n terminates. A loop that never updates its counter does not — it is not an algorithm.

---

### 2.2 Definiteness

Every step of the algorithm must be **precisely and unambiguously defined**.

There must be no room for interpretation. Each instruction must have exactly one meaning and must be clear enough to be carried out by any executor — human or machine.

**Example:** "Add 1 to x" is definite. "Do something with x" is not.

---

### 2.3 Input

An algorithm has **zero or more inputs** — quantities that are given before the algorithm begins, taken from a specified set.

Zero input is valid. For example, an algorithm that always generates the first 10 prime numbers requires no input.

---

### 2.4 Output

An algorithm must produce **one or more outputs** — quantities that have a defined relationship to the inputs.

The output is the result or answer the algorithm is supposed to produce.

**Example:** A sorting algorithm takes an array as input and produces a sorted array as output.

---

### 2.5 Effectiveness

Every step of the algorithm must be **basic enough to be carried out exactly** and in a finite amount of time.

Operations must be simple, executable, and not just theoretically possible.

**Example:** "Multiply two numbers" is effective. "Find the largest prime number" is not effective because no such largest prime exists.

---

## 3. Algorithm vs Program vs Flowchart

These three terms are often confused. The table below distinguishes them clearly:

| Aspect | Algorithm | Program | Flowchart |
|---|---|---|---|
| Definition | Step-by-step logic to solve a problem | Implementation of an algorithm in a programming language | Graphical/visual representation of an algorithm |
| Language | Natural language or pseudocode | Python, C, Java, etc. | Symbols and arrows |
| Machine dependency | Independent | Dependent on language and machine | Independent |
| Terminates | Must always terminate | May or may not | Represents termination with an oval symbol |
| Purpose | To describe the solution logic | To execute on a computer | To visualize the flow of logic |

---

**Key point to remember:**

- An **algorithm** is the idea.
- A **program** is the implementation of that idea.
- A **flowchart** is a picture of that idea.

The same algorithm can be implemented as a program in any language, and drawn as a flowchart — the underlying logic remains identical.

---

## 4. Pseudocode

### 4.1 What is Pseudocode

Pseudocode is an **informal, human-readable description of an algorithm** that uses the structural conventions of programming but is not tied to any specific language. It sits between natural language and actual code.

It is used to plan and communicate algorithms clearly before writing actual code.

---

### 4.2 Why Use Pseudocode

- It focuses on logic, not syntax.
- It is readable by anyone regardless of their programming language background.
- It is faster to write than actual code during design.
- It is the standard format used in DAA textbooks (including CLRS) and university exams.

---

### 4.3 Pseudocode Conventions

| Element | Convention |
|---|---|
| Assignment | `x = value` |
| Comparison | `==`, `!=`, `<`, `>`, `<=`, `>=` |
| Loop (count-based) | `for i = 1 to n` |
| Loop (condition-based) | `while condition do` |
| Conditional | `if condition then ... else ...` |
| Array access | `A[i]` (1-indexed unless stated otherwise) |
| Function/procedure | `FUNCTION-NAME(parameters)` |
| Return value | `return value` |
| End of block | `end for`, `end while`, `end if` |
| Comments | `// comment text` |

---

### 4.4 Pseudocode Example 1 — Linear Search

**Problem:** Given an array A of n elements and a target value x, find the index of x in A. If x is not present, return -1.

```
LINEAR-SEARCH(A, n, x)

1.  for i = 1 to n
2.      if A[i] == x
3.          return i        // element found, return its index
4.      end if
5.  end for
6.  return -1               // element not found
```

**Trace through an example:**

Let A = [4, 2, 7, 1, 9] and x = 7

- i = 1: A[1] = 4, not equal to 7
- i = 2: A[2] = 2, not equal to 7
- i = 3: A[3] = 7, equal to 7 — return 3

Output: 3

---

### 4.5 Pseudocode Example 2 — Find Maximum Element

**Problem:** Given an array A of n elements, find the largest element.

```
FIND-MAX(A, n)

1.  max = A[1]              // assume first element is the maximum
2.  for i = 2 to n
3.      if A[i] > max
4.          max = A[i]      // update maximum
5.      end if
6.  end for
7.  return max
```

**Trace through an example:**

Let A = [3, 8, 1, 6, 2]

- max = 3 (initial)
- i = 2: A[2] = 8 > 3, so max = 8
- i = 3: A[3] = 1, not > 8
- i = 4: A[4] = 6, not > 8
- i = 5: A[5] = 2, not > 8

Output: 8

---

### 4.6 Pseudocode Example 3 — Bubble Sort (Preview)

**Problem:** Sort an array A of n elements in ascending order.

```
BUBBLE-SORT(A, n)

1.  for i = 1 to n - 1
2.      for j = 1 to n - i
3.          if A[j] > A[j+1]
4.              temp = A[j]
5.              A[j] = A[j+1]
6.              A[j+1] = temp   // swap
7.          end if
8.      end for
9.  end for
```

This example is given to show how a more complex algorithm is still readable in pseudocode. The analysis of Bubble Sort's time complexity will be covered in Lecture 6.

## 5. Important Definitions

**Algorithm:** A finite, definite, effective sequence of steps that takes zero or more inputs and produces one or more outputs to solve a computational problem.

**Finiteness:** The property that an algorithm must terminate after a finite number of steps under all conditions.

**Definiteness:** The property that every step of an algorithm must be precisely and unambiguously defined.

**Effectiveness:** The property that every operation in an algorithm must be basic, executable, and completable in finite time.

**Pseudocode:** An informal notation for describing algorithms using structured English and programming-like constructs, independent of any specific programming language.

**Program:** A concrete implementation of an algorithm in a specific programming language, intended for execution on a machine.

---

## 6. Summary 

- An algorithm is a finite, unambiguous, effective sequence of steps to solve a problem.
- Every algorithm must satisfy five properties: Finiteness, Definiteness, Input, Output, Effectiveness.
- An algorithm is the logic; a program is its implementation; a flowchart is its visual representation.
- Pseudocode is the standard way to write algorithms in this course — it focuses on logic, not syntax.
- Pseudocode must be written clearly enough that anyone can trace through it step by step.
