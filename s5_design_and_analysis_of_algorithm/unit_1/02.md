## 1. Sets

### What is a Set?

A **set** is an **unordered collection of distinct objects** (called elements or members).
- Order does not matter: `{1, 2, 3}` = `{3, 1, 2}`
- No duplicates: `{1, 1, 2}` is not a valid set — it simplifies to `{1, 2}`

```
A = {1, 2, 3, 4, 5}       ← set of five integers
B = {a, e, i, o, u}        ← set of vowels
C = {}  or  ∅              ← empty set
```

| Notation | Meaning | Example |
|---|---|---|
| `x ∈ A` | x **is** an element of A | `3 ∈ {1, 2, 3}` → TRUE |
| `x ∉ A` | x **is not** an element of A | `5 ∉ {1, 2, 3}` → TRUE |

---

### Set Operations

> Let **A = {1, 2, 3}** and **B = {3, 4, 5}** for all examples below.

| Operation | Symbol | Rule | Result |
|---|---|---|---|
| Union | A ∪ B | Everything in A **or** B | `{1, 2, 3, 4, 5}` |
| Intersection | A ∩ B | Only what's in **both** | `{3}` |
| Difference | A − B | In A but **not** in B | `{1, 2}` |
| Difference | B − A | In B but **not** in A | `{4, 5}` |
| Complement | A' | Everything **not** in A (within universal set U) | `U − A` |

**Memory tip:** Think of Union as "OR", Intersection as "AND."

---

### Special Sets and Concepts

| Concept | Symbol | Description | Example |
|---|---|---|---|
| Empty Set | ∅ | No elements at all | `{}` |
| Universal Set | U | All elements under consideration | Depends on context |
| Subset | A ⊆ B | Every element of A is also in B | `{1,2} ⊆ {1,2,3}` |
| Proper Subset | A ⊂ B | A ⊆ B **and** A ≠ B | `{1,2} ⊂ {1,2,3}` |
| Power Set | P(A) | Set of **all** subsets of A | See below |
| Cardinality | \|A\| | Number of elements in A | `\|{a,b,c}\| = 3` |

#### Power Set — Worked Example

```
A = {1, 2}

P(A) = { {}, {1}, {2}, {1,2} }
         ↑     ↑    ↑     ↑
       empty  one  one  both
       set   elem elem  elems

|P(A)| = 2^|A| = 2^2 = 4
```

> **Formula:** If |A| = n, then |P(A)| = **2ⁿ**

| \|A\| | \|P(A)\| | All subsets |
|---|---|---|
| 0 | 1 | `{}` |
| 1 | 2 | `{}, {a}` |
| 2 | 4 | `{}, {a}, {b}, {a,b}` |
| 3 | 8 | 8 subsets |
| 10 | 1024 | — |

---

### Where Sets Appear in DAA

- **Graphs (Unit 4):** A graph is defined as G = (V, E) — V is the set of vertices, E is the set of edges.
- **NP-Completeness (Unit 5):** The classes P and NP are sets of decision problems.
- **Greedy Algorithms (Unit 3):** Activity selection picks the best **subset** of non-overlapping activities.

## 2. Relations

### What is a Relation?

Think of a relation as a **rule that connects elements of one set to elements of another**.

In everyday language:
- "is taller than" is a relation between people
- "is a factor of" is a relation between numbers
- "is connected to" is a relation between cities

Formally: a **relation R** from set A to set B is a **subset of the Cartesian product A × B**.

---

**Step 1 — Understand Cartesian Product first.**

The Cartesian product A × B is simply **every possible pairing** of one element from A with one element from B.

```
A = {1, 2}
B = {x, y}

A × B = { (1,x), (1,y), (2,x), (2,y) }
         ↑ every element of A paired with every element of B
```

---

**Step 2 — A relation just picks some of those pairs.**

Not all pairs need to be included — the relation selects only the pairs where the rule holds.

```
A = {1, 2, 3}
Rule: "a R b  if  a < b"

All possible pairs from A × A:
  (1,1) (1,2) (1,3)
  (2,1) (2,2) (2,3)
  (3,1) (3,2) (3,3)

Which pairs satisfy a < b?
  (1,2) ✓  because 1 < 2
  (1,3) ✓  because 1 < 3
  (2,3) ✓  because 2 < 3

So R = { (1,2), (1,3), (2,3) }
```

If **(a, b) ∈ R**, we write **a R b** and say "a is related to b."

---

### Properties of Relations

> **Note:** The three properties below apply only when a relation R is defined **on a single set A**, meaning R ⊆ A × A. Both elements of every pair (a, b) come from the same set A — unlike the general case where a ∈ A and b ∈ B from two different sets.

---

#### 1. Reflexive

**Condition:** ∀ a ∈ A, (a, a) ∈ R

Every element of A must be related to itself. The pair (a, a) must exist in R for every single element a in the set.

```
Let A = {1, 2, 3}

For R to be reflexive, ALL of these pairs must be in R:
  (1,1), (2,2), (3,3)

Even if (1,2), (2,3) etc. are present,
if even ONE of (1,1) or (2,2) or (3,3) is missing → NOT reflexive.
```

**Example 1 — R defined by "a ≤ b" on A = {1, 2, 3}**

```
R = { (1,1), (1,2), (1,3), (2,2), (2,3), (3,3) }

Check (a, a) for each element:
  (1,1) ∈ R  →  1 ≤ 1  ✓
  (2,2) ∈ R  →  2 ≤ 2  ✓
  (3,3) ∈ R  →  3 ≤ 3  ✓

∀ a ∈ A, (a,a) ∈ R  →  R is REFLEXIVE ✓
```

**Example 2 — R defined by "a < b" on A = {1, 2, 3}**

```
R = { (1,2), (1,3), (2,3) }

Check (a, a) for each element:
  (1,1) ∈ R?  →  1 < 1  ✗
  (2,2) ∈ R?  →  2 < 2  ✗
  (3,3) ∈ R?  →  3 < 3  ✗

(a, a) ∉ R for any a  →  R is NOT REFLEXIVE ✗
```

---

#### 2. Symmetric

**Condition:** ∀ a, b ∈ A, if (a, b) ∈ R then (b, a) ∈ R

For every pair that exists in R, the reversed pair must also exist in R.

```
Let A = {1, 2, 3}

If (1, 2) ∈ R  →  (2, 1) must also ∈ R
If (2, 3) ∈ R  →  (3, 2) must also ∈ R

Every pair must have its reverse also present in R.
```

**Example 1 — R = {(1,2), (2,1), (2,3), (3,2)} on A = {1, 2, 3}**

```
Check each pair and its reverse:
  (1,2) ∈ R  →  (2,1) ∈ R  ✓
  (2,1) ∈ R  →  (1,2) ∈ R  ✓
  (2,3) ∈ R  →  (3,2) ∈ R  ✓
  (3,2) ∈ R  →  (2,3) ∈ R  ✓

Every pair has its reverse  →  R is SYMMETRIC ✓
```

**Example 2 — R defined by "a < b" on A = {1, 2, 3}**

```
R = { (1,2), (1,3), (2,3) }

  (1,2) ∈ R  →  is (2,1) ∈ R?  2 < 1 is false  →  (2,1) ∉ R  ✗

One pair fails  →  R is NOT SYMMETRIC ✗
```

---

#### 3. Transitive

**Condition:** ∀ a, b, c ∈ A, if (a, b) ∈ R and (b, c) ∈ R, then (a, c) ∈ R

If the pair (a, b) exists and the pair (b, c) exists, then the pair (a, c) must also exist — the shared middle element b creates an obligation.

```
Let A = {1, 2, 3, 4}

If (1,2) ∈ R  and  (2,3) ∈ R  →  (1,3) must ∈ R
If (2,3) ∈ R  and  (3,4) ∈ R  →  (2,4) must ∈ R
If (1,3) ∈ R  and  (3,4) ∈ R  →  (1,4) must ∈ R
```

**Example 1 — R defined by "a < b" on A = {1, 2, 3}**

```
R = { (1,2), (1,3), (2,3) }

Find all (a,b) and (b,c) pairs and check if (a,c) exists:
  (1,2) ∈ R and (2,3) ∈ R  →  is (1,3) ∈ R?  YES ✓

No other combinations exist where the middle element matches.

All chains satisfied  →  R is TRANSITIVE ✓
```

**Example 2 — R = {(1,2), (2,3)} on A = {1, 2, 3}**

```
  (1,2) ∈ R and (2,3) ∈ R  →  is (1,3) ∈ R?  NO ✗

(1,3) is missing  →  R is NOT TRANSITIVE ✗
```

**Key point:** Transitivity only checks pairs that share a middle element. If no such chain (a,b)+(b,c) exists in R at all, the condition is **vacuously satisfied** — R is transitive by default.

---

### Equivalence Relation

A relation that satisfies **all three** — reflexive, symmetric, and transitive — is called an **equivalence relation**.

Equivalence relations are powerful because they **group elements into clusters** called equivalence classes, where every element in a cluster is "equivalent" to every other.

---

**Classic Example: Modular Arithmetic — "same remainder when divided by 3"**

Two numbers are related if dividing them by 3 gives the same remainder.

```
Numbers grouped by remainder when divided by 3:

  Remainder 0:  0, 3, 6, 9, 12 ...   → all equivalent to each other
  Remainder 1:  1, 4, 7, 10, 13 ...  → all equivalent to each other
  Remainder 2:  2, 5, 8, 11, 14 ...  → all equivalent to each other
```

Now verify all three properties:

```
REFLEXIVE — does every number relate to itself?
  Is 5 ≡ 5 (mod 3)?
  5 ÷ 3 = remainder 2,  5 ÷ 3 = remainder 2  → same ✓

SYMMETRIC — if a relates to b, does b relate to a?
  Is 5 ≡ 8 (mod 3)?   5 → remainder 2,  8 → remainder 2  ✓
  Is 8 ≡ 5 (mod 3)?   8 → remainder 2,  5 → remainder 2  ✓
  Both directions hold ✓

TRANSITIVE — if a~b and b~c, does a~c hold?
  2 ≡ 5 (mod 3)?   2 → rem 2,  5 → rem 2  ✓
  5 ≡ 8 (mod 3)?   5 → rem 2,  8 → rem 2  ✓
  2 ≡ 8 (mod 3)?   2 → rem 2,  8 → rem 2  ✓  chain holds
```

All three hold → **"same remainder mod 3" is an equivalence relation** ✓

---

### Quick Property Comparison

| Relation | Plain meaning | Reflexive | Symmetric | Transitive | Type |
|---|---|---|---|---|---|
| `a = b` | exact equality | ✓ | ✓ | ✓ | Equivalence |
| `a ≡ b (mod m)` | same remainder | ✓ | ✓ | ✓ | Equivalence |
| `a ≤ b` | less or equal | ✓ | ✗ | ✓ | Partial order |
| `a < b` | strictly less | ✗ | ✗ | ✓ | Strict order |
| "is sibling of" | shared parents | ✗ | ✓ | ✓ | — |
| "is parent of" | one level up | ✗ | ✗ | ✗ | — |
| "is ancestor of" | any level up | ✗ | ✗ | ✓ | — |

---

### Where Relations Appear in DAA

- **Sorting:** Comparison-based sorts rely on a total ordering relation (e.g., `≤`) on elements.
- **Graph reachability:** "u can reach v" defines a relation. If it's transitive, we get the transitive closure — the foundation of Floyd-Warshall (Unit 4).
- **Kruskal's Algorithm (Unit 3):** Uses Union-Find, which tracks equivalence classes of connected components.

## 3. Functions

### What is a Function?

A **function** f : A → B is a **special relation** where every element of A maps to **exactly one** element of B.

```
f : A → B
    ↑   ↑
  domain  co-domain
```

| Term | Meaning |
|---|---|
| Domain (A) | The set of all valid inputs |
| Co-domain (B) | The set of all possible outputs |
| Range | The set of outputs actually produced (range ⊆ co-domain) |

**Example:**
```
f(x) = x²    with A = {1, 2, 3},  B = {1, 4, 9, 16}

f(1) = 1,  f(2) = 4,  f(3) = 9

Range = {1, 4, 9}   ← not all of B is covered (16 is unused)
```

---

### Types of Functions

#### Injective (One-One)
No two inputs map to the **same output**. Each output is used at most once.

```
Condition: f(a) = f(b)  →  a = b

✓ Injective:              ✗ Not Injective:
  1 → A                    1 → A
  2 → B                    2 → A   ← two inputs → same output
  3 → C                    3 → B
```

> **Real example:** f(x) = 2x is injective — 2×3 ≠ 2×4, always.

#### Surjective (Onto)
Every element in the co-domain is **hit** by at least one input.

```
Condition: For every b ∈ B, ∃ a ∈ A such that f(a) = b

✓ Surjective:             ✗ Not Surjective:
  1 → A                    1 → A
  2 → B                    2 → B
  3 → C                    3 → B   ← C is never reached!
```

#### Bijective (One-One AND Onto)
A perfect pairing — every input maps to a unique output, and every output is used exactly once.

```
✓ Bijective:
  1 → A
  2 → B     ← perfect 1-to-1 correspondence
  3 → C
```

---

#### Summary Table

| Property | Every output used? | Every input → unique output? | Key example |
|---|---|---|---|
| Injective (One-One) | Not necessarily | ✓ Yes | f(x) = 2x |
| Surjective (Onto) | ✓ Yes | Not necessarily | f: {1,2,3} → {a,b,c} (all covered) |
| Bijective | ✓ Yes | ✓ Yes | f(x) = x + 1 on integers |

---

### Where Functions Appear in DAA

| Use case | Type of function | Why it matters |
|---|---|---|
| Hash functions | Ideally injective | Avoids collisions (two keys → same slot) |
| Complexity analysis | T(n), f(n) | Maps input size → time cost |
| Logarithm `log₂(n)` | Strictly increasing | Appears in binary search, merge sort, trees |
| Exponent `2ⁿ` | Strictly increasing | Appears in brute-force/NP analysis |

## 4. Vectors and Matrices

### Vectors

A **vector** is an **ordered list of numbers** — a 1-dimensional structure.

```
Row vector:     v = [1,  3,  5]

Column vector:  v = | 1 |
                    | 3 |
                    | 5 |
```

In DAA, vectors represent things like: a single row of an adjacency matrix, a list of edge weights, or one layer of a dynamic programming table.

---

### What is a Matrix?

A **matrix** is a **rectangular array of numbers** arranged in rows and columns.
An **m × n matrix** has m rows and n columns.

```
      col1 col2 col3
row1 | 1    2    3  |
row2 | 4    5    6  |    →  a 2×3 matrix
```

Element at row i, column j → **A[i][j]**

```
Example: A = | 1  2  3 |
             | 4  5  6 |
             | 7  8  9 |    (3×3 matrix)

A[2][3] = 6   (row 2, column 3)
```

---

### Matrix Addition

Matrices of the **same dimensions** are added element-by-element.

```
| 1  2 |   | 5  6 |   | 1+5   2+6  |   | 6    8  |
| 3  4 | + | 7  8 | = | 3+7   4+8  | = | 10  12  |
```

**Rule:** C[i][j] = A[i][j] + B[i][j]

**Time complexity of adding two n×n matrices:** O(n²)

---

### Matrix Multiplication

To multiply A (size m×n) with B (size n×p):
- Number of **columns in A** must equal number of **rows in B**
- Result C has size **m×p**

**Rule:** `C[i][j] = Σ A[i][k] × B[k][j]`  for k = 1 to n

#### Worked Example — 2×2 Multiplication

```
A = | 1  2 |     B = | 5  6 |
    | 3  4 |         | 7  8 |
```

Computing each cell of C = A × B:

```
C[1][1] = (1×5) + (2×7) =  5 + 14 = 19
C[1][2] = (1×6) + (2×8) =  6 + 16 = 22
C[2][1] = (3×5) + (4×7) = 15 + 28 = 43
C[2][2] = (3×6) + (4×8) = 18 + 32 = 50

Result: C = | 19  22 |
            | 43  50 |
```

**Time complexity:** Multiplying two n×n matrices → **O(n³)**

> **Why this matters:** Strassen's Algorithm (Unit 2) reduces this to **O(n^2.81)** using divide-and-conquer — one of the most famous algorithmic improvements in computer science.

---

### Dimension Compatibility Table

| A (size) | B (size) | Valid? | Result (size) |
|---|---|---|---|
| 2×3 | 3×4 | ✓ | 2×4 |
| 3×2 | 3×4 | ✗ | — (inner dims 2≠3) |
| 4×4 | 4×4 | ✓ | 4×4 |
| 1×n | n×1 | ✓ | 1×1 (a scalar) |

---

### Identity Matrix

The **identity matrix I** is a square matrix with **1s on the diagonal** and **0s everywhere else**.

```
I₃ = | 1  0  0 |
     | 0  1  0 |
     | 0  0  1 |
```

**Key property:** A × I = I × A = A *(same as multiplying a number by 1)*

---

### Transpose of a Matrix

The **transpose** A^T is obtained by **swapping rows and columns**.

```
A = | 1  2  3 |       A^T = | 1  4 |
    | 4  5  6 |              | 2  5 |
                             | 3  6 |

Rule: A^T[i][j] = A[j][i]
```

---

### Adjacency Matrix — Graph Representation

A graph with n vertices is represented as an n×n matrix where:
- **A[i][j] = 1** if there is an edge from vertex i to vertex j
- **A[i][j] = 0** otherwise

```
Graph:  1 — 2
        |   |
        3 — 4

Adjacency Matrix:
       1  2  3  4
    1 [0, 1, 1, 0]
    2 [1, 0, 0, 1]
    3 [1, 0, 0, 1]
    4 [0, 1, 1, 0]
```

| Matrix Chain Multiplication (Unit 4) | Classic dynamic programming problem |

## 5. Linear Equations and Inequalities

### Linear Equations

A **linear equation** in variables x₁, x₂, ..., xₙ:

```
a₁x₁ + a₂x₂ + ... + aₙxₙ = b
```

A **system** of linear equations must all be satisfied simultaneously.

#### Worked Example

```
System:   2x + 3y = 8   ...(i)
           x  −  y = 1   ...(ii)

From (ii):  x = y + 1
Sub into (i): 2(y+1) + 3y = 8
              5y = 6
              y = 6/5,   x = 11/5
```

---

### Linear Inequalities

Same structure but with an inequality sign:

```
a₁x₁ + a₂x₂ + ... + aₙxₙ  ≤  b
```

A system of inequalities defines a **feasible region** — the set of all points satisfying all constraints.

#### Example — Feasible Region

```
Constraints:
  x + y ≤ 10
  x     ≥ 2
      y ≥ 3

All three must hold simultaneously.
E.g., (x=3, y=4) satisfies all three  ✓
      (x=1, y=4) fails x ≥ 2          ✗
```

---

### Where These Appear in DAA

| DAA Topic | Constraint type |
|---|---|
| Knapsack problem (Unit 3) | `Σ wᵢxᵢ ≤ W` (capacity constraint) |
| Network flow (Unit 4) | Flow ≤ capacity on each edge |
| LP relaxation of NP problems (Unit 5) | Linear inequalities define feasible sets |
| Greedy correctness proofs | Show solution stays within constraint boundaries |

## 6. Key Definitions — Quick Reference

| Term | Definition |
|---|---|
| **Set** | Unordered collection of distinct objects |
| **Union A ∪ B** | All elements in A or B or both |
| **Intersection A ∩ B** | Elements in both A and B |
| **Power Set P(A)** | All subsets of A; \|P(A)\| = 2ⁿ |
| **Relation** | Subset of A × B describing a pairing |
| **Reflexive** | (a, a) ∈ R for all a |
| **Symmetric** | (a, b) ∈ R implies (b, a) ∈ R |
| **Transitive** | (a,b) and (b,c) in R implies (a,c) ∈ R |
| **Equivalence Relation** | Reflexive + Symmetric + Transitive |
| **Function f : A → B** | Every a ∈ A maps to exactly one b ∈ B |
| **Injective** | No two inputs → same output |
| **Surjective** | Every output is mapped to by at least one input |
| **Bijective** | Both injective and surjective |
| **Matrix** | Rectangular array of numbers; m×n dimensions |
| **Matrix Multiplication** | C[i][j] = Σ A[i][k]×B[k][j]; O(n³) cost |
| **Identity Matrix** | Diagonal = 1, rest = 0; A×I = A |
| **Linear Inequality** | a₁x₁ + ... + aₙxₙ ≤ b |

```
SETS        →  unordered, distinct; operations: ∪ ∩ −; |P(A)| = 2ⁿ

MATRICES    →  rectangular number arrays; multiplication is O(n³) → Strassen improves this
INEQUALITIES→  define feasible regions; appear in greedy, DP, and NP topics
```
