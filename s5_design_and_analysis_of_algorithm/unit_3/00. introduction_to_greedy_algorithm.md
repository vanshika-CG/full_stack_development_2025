# Greedy Algorithms


# 1. Introduction

In Design and Analysis of Algorithms (DAA), there are many techniques used to solve computational problems efficiently. Some of the most popular techniques are:

- Brute Force
- Divide and Conquer
- Dynamic Programming
- Greedy Algorithm
- Backtracking
- Branch and Bound

Among these, the **Greedy Algorithm** is one of the simplest and most efficient approaches for solving optimization problems.

Unlike Divide and Conquer, which divides a problem into smaller sub-problems, or Dynamic Programming, which stores previously computed results, the Greedy Algorithm makes the **best possible choice at the current step** without worrying about future consequences.

The idea behind a Greedy Algorithm is very simple.

> **"Always choose the option that appears to be the best at the current moment."**

Because of this behavior, it is called a **Greedy Algorithm**.

Many real-world problems such as job scheduling, network routing, data compression, minimum spanning tree, shortest path, and activity scheduling are solved using greedy techniques.


# 2. What is a Greedy Algorithm?

A Greedy Algorithm is an algorithmic strategy that builds the solution step by step.

At every step, it selects the **locally optimal** (best immediate) choice, hoping that these local choices will eventually lead to the **globally optimal** (best overall) solution.

Unlike Dynamic Programming, a Greedy Algorithm **never revisits or changes a previous decision**.

Once a choice is made, it becomes permanent.


## Formal Definition

> **Greedy Algorithm:**  
> A Greedy Algorithm is an algorithmic paradigm that constructs a solution incrementally by repeatedly making the locally optimal choice at each step, with the hope of obtaining a globally optimal solution.


# 3. Why is it Called "Greedy"?

Imagine a child standing in front of a box full of chocolates.

Instead of thinking,

> "If I take this chocolate now, will I get a bigger chocolate later?"

the child simply picks the **largest chocolate available at that moment**.

The child does not consider future possibilities.

He always chooses the **best immediate option**.

This behavior is called **greedy behavior**.

Similarly,

a Greedy Algorithm always chooses the best available option at the current step without reconsidering earlier decisions.


# 4. Real-Life Example of Greedy Algorithm

Suppose you are traveling from your home to college.

There are three possible routes.

```
Route A

Time = 25 Minutes
```

```
Route B

Time = 35 Minutes
```

```
Route C

Time = 40 Minutes
```

Without knowing the traffic conditions later,

you naturally choose

```
Route A
```

because it is the shortest **right now**.

This is a Greedy decision.

You did not analyze every possible future scenario.

You simply selected the best available option.


# 5. General Characteristics of Greedy Algorithms

Every Greedy Algorithm follows some common characteristics.

These characteristics help us identify whether a problem can be solved using the Greedy approach.


## 5.1 Makes the Best Local Choice

A Greedy Algorithm always chooses the best option available at the current step.

It never postpones the decision.

### Example

Suppose you have coins

```
₹10

₹5

₹2

₹1
```

You need to make

```
₹18
```

The Greedy Algorithm chooses

```
₹10
```

Remaining amount

```
₹8
```

Then chooses

```
₹5
```

Remaining amount

```
₹3
```

Then

```
₹2
```

Remaining

```
₹1
```

Finally

```
₹1
```

Total coins

```
10 + 5 + 2 + 1
```

The algorithm always selected the largest possible coin first.


## 5.2 Builds the Solution Step by Step

A Greedy Algorithm never computes the complete solution at once.

Instead,

it gradually builds the final solution.

```
Start

↓

Choose Best Option

↓

Update Solution

↓

Choose Next Best Option

↓

Repeat

↓

Final Solution
```


## 5.3 Never Changes Previous Decisions

One of the most important properties of a Greedy Algorithm is that once a decision is made,

it is **never changed**.

For example,

suppose you selected

```
Activity A
```

The algorithm will never remove Activity A later.

This is different from Dynamic Programming, where previous decisions may influence future computations.


## 5.4 Solves Optimization Problems

Greedy Algorithms are mainly used for optimization.

Optimization means

- Finding the maximum value, or
- Finding the minimum value.

Examples include:

- Minimum cost
- Maximum profit
- Minimum distance
- Maximum number of activities
- Minimum spanning tree
- Shortest path


## 5.5 Fast and Efficient

Most Greedy Algorithms are faster than Dynamic Programming algorithms because they avoid solving every possible sub-problem.

Many Greedy Algorithms have time complexities such as

```
O(n)

O(n log n)
```

making them suitable for large datasets.


## 5.6 Requires Greedy Choice Property

A problem can be solved using a Greedy Algorithm **only if** making the locally optimal choice always leads to the globally optimal solution.

If this property does not hold,

the Greedy Algorithm may produce an incorrect answer.


## 5.7 Requires Optimal Substructure

The optimal solution of the whole problem must contain the optimal solutions of its smaller sub-problems.

Without this property,

the Greedy approach cannot guarantee the correct answer.


# 6. Elements of Greedy Strategy

A Greedy Algorithm does not simply pick a random option.

It follows a well-defined strategy to make decisions.

Before a Greedy Algorithm can solve any problem, it must answer several important questions.

- What are all the possible choices?
- Which choice should be selected?
- Is the selected choice valid?
- What is the final goal?
- When should the algorithm stop?

These questions form the **Elements of Greedy Strategy**.

To understand these elements, let us consider a simple real-life example.


## Example Scenario

Suppose you want to travel from your **Home** to your **School**.

There are multiple roads available.

```
                    School
                       ▲
                       │
              2 km     │
                       │
               Road D  │
                       │
Home ─── Road A ─── Junction
 │                     ▲
 │                     │
 │                     │
 │                     │
Road B              Road C
 │                     │
 │                     │
Market             Railway Station
```

Assume the distances are

```
Road A = 3 km

Road B = 5 km

Road C = 4 km

Road D = 2 km
```

Your objective is to reach school using the **shortest possible distance**.

Now let us understand how every element of the Greedy Strategy works in this example.


## 6.1 Candidate Set

The **Candidate Set** contains all the possible choices available at the current step.

In our example,

when you leave your home,

you can choose any available road.

```
Candidate Set

↓

Road A

Road B
```

After reaching the junction,

the available roads become

```
Candidate Set

↓

Road C

Road D
```

Therefore,

the Candidate Set keeps changing as you move from one location to another.

> **Definition:**  
> The Candidate Set is the collection of all possible choices from which the algorithm can select the next candidate.


## 6.2 Selection Function

The **Selection Function** decides which candidate should be selected from the Candidate Set.

Since our objective is to travel the minimum distance,

the Greedy Algorithm always selects the **shortest available road**.

Suppose

```
Road A = 3 km

Road B = 5 km
```

The algorithm compares both roads.

```
3 km < 5 km
```

Therefore,

```
Choose

Road A
```

The Selection Function always chooses the **best immediate option**.

> **Definition:**  
> The Selection Function selects the best candidate according to the optimization objective.


## 6.3 Feasibility Function

Selecting the shortest road alone is not enough.

The algorithm must also verify whether the selected road is **valid**.

Suppose

```
Road A

↓

Closed due to construction
```

Although it is the shortest road,

it cannot be used.

Therefore,

the Greedy Algorithm rejects it and selects another road.

Similarly,

suppose

```
Road D

↓

Flooded
```

Again,

the road cannot be selected.

The Feasibility Function ensures that every selected candidate satisfies all problem constraints.

> **Definition:**  
> The Feasibility Function checks whether the selected candidate can be included in the solution without violating any constraints.


## 6.4 Objective Function

The **Objective Function** specifies what we want to achieve.

Different problems have different objectives.

Some common objectives are

```
Minimize Cost

Maximize Profit

Minimize Time

Minimize Distance

Maximize Number of Activities
```

In our example,

the objective is

```
Minimize Total Distance
```

Every decision made by the algorithm moves towards achieving this goal.

> **Definition:**  
> The Objective Function defines the optimization goal that the algorithm tries to achieve.


## 6.5 Solution Function

The algorithm must know when the solution is complete.

In our example,

the solution is complete when

```
Current Location

=

School
```

Until then,

the algorithm continues selecting roads.

```
Home

↓

Road A

↓

Road D

↓

School

✓ Stop
```

Once the destination is reached,

the algorithm terminates.

> **Definition:**  
> The Solution Function determines whether a complete solution has been obtained.


## Putting Everything Together

Suppose the available roads are

```

↓

Road A (3 km)

Road B (5 km)
```

The algorithm works as follows.

```
Candidate Set

↓

Road A

Road B
```

Selection Function

```
Choose

Road A
```

Feasibility Function

```
Road A Open?

```

Objective Function

```

```

Continue.

```

```

Again,

```
Candidate Set

↓

Road C

Road D
```

Selection Function

```
Choose

Road D
```

Feasibility Function

```
Road D Open?

↓

Yes
```

Objective Function

```
Still minimizing distance
```

Finally,

```
Reach School

↓

Solution Complete

↓

```

This is exactly how a Greedy Algorithm makes decisions step by step.

At every stage,

it

- identifies all possible choices,
- selects the best one,
- verifies whether it is valid,
- moves towards the optimization objective, and
- stops when the destination is reached.

## 6.6 Solution Function

The Solution Function determines whether the final solution has been obtained.

If yes,

the algorithm terminates.

Otherwise,

it continues selecting more candidates.


## 6.7 Greedy Choice Property

The Greedy Choice Property states that

> **A globally optimal solution can be obtained by making a locally optimal choice at each step.**

This is the most important property of every Greedy Algorithm.

Without this property,

the Greedy approach cannot guarantee the correct solution.


## 6.8 Optimal Substructure

A problem exhibits Optimal Substructure if

the optimal solution of the whole problem can be constructed from the optimal solutions of its smaller sub-problems.

This property is shared by both

- Greedy Algorithms
- Dynamic Programming

However,

their methods of solving the problem are different.


# 7. Summary

- Greedy Algorithms make the best immediate choice at every step.
- Once a decision is taken, it is never changed.
- They are mainly used for optimization problems.
- They are generally faster than Dynamic Programming.
- A Greedy Algorithm works correctly only when the problem satisfies the **Greedy Choice Property** and **Optimal Substructure**.
- The main elements of a Greedy Strategy are Candidate Set, Selection Function, Feasibility Function, Objective Function, Solution Function, Greedy Choice Property, and Optimal Substructure.
