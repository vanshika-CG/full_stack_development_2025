# Minimum Spanning Tree (MST) and Kruskal's Algorithm


# 1. Introduction

In many real-world situations, we need to connect multiple locations while spending the **least possible cost**.

For example,

- Connecting cities using roads.
- Connecting computers in a network using cables.
- Providing electricity to villages.
- Laying water pipelines.
- Connecting mobile towers using optical fiber.

In all these situations, our goal is **not just to connect all locations**, but also to **minimize the total construction cost**.

This type of problem is solved using a concept called the **Minimum Spanning Tree (MST)**.

The Minimum Spanning Tree is one of the most important applications of the **Greedy Algorithm** in Graph Theory.

Among the different algorithms used to find an MST, **Kruskal's Algorithm** is one of the simplest and most widely used.

Before understanding Kruskal's Algorithm, we must first understand some basic concepts related to graphs.


# 2. What is a Graph?

A **Graph** is a data structure that consists of a collection of **vertices (nodes)** and **edges (connections)**.

- **Vertices (V):** Represent objects or locations.
- **Edges (E):** Represent the connection between two vertices.

For example,

suppose we have four cities.

```
Ahmedabad

Surat

Vadodara

Rajkot
```

If roads connect these cities, then

- Each city becomes a **Vertex**.
- Each road becomes an **Edge**.

Graph representation:

```
          Ahmedabad
          /       \
         /         \
      Surat------Vadodara
          \
           \
          Rajkot
```

Here,

```
Vertices = Cities

Edges = Roads
```


# 3. What is a Weighted Graph?

Sometimes every edge has a numerical value associated with it.

This value is called the **Weight**.

The weight may represent

- Distance
- Cost
- Time
- Cable Length
- Fuel Consumption

Example:

```
          A
       4 / \ 2
        /   \
       B-----C
         3
```

Here,

```

A → C = 2 km

B → C = 3 km
```

This is called a **Weighted Graph** because every edge has a weight.


# 4. What is a Tree?

A **Tree** is a special type of graph that satisfies two important properties:

1. All vertices are connected.
2. There are **no cycles (loops)**.

Example:

```
      A
     / \
    B   C
       / \
      D   E
```

This is a Tree because

- Every node is connected.
- No cycle exists.


## What is a Cycle?

A **Cycle** is a path that starts and ends at the same vertex without repeating an edge.

Example:

```

|        |

```

One possible path is

```

↓

```

Since we returned to the starting vertex,

this graph contains a **Cycle**.

A Tree **never** contains a cycle.

## Definition

> A **Spanning Tree** is a subgraph of a connected graph that contains **all the vertices** and connects them using exactly **V − 1 edges**, without forming any cycle.


### Property 4

If a cycle exists,

A graph may have **many different spanning trees**.

Our goal is to find the spanning tree whose **total edge weight is the smallest**.

This tree is called the **Minimum Spanning Tree (MST).**

# 7. Why Do We Need a Minimum Spanning Tree?

Suppose a government wants to connect four villages using electricity cables.

The distances are

```
Village A ↔ Village B = 8 km

Village A ↔ Village C = 3 km

Village B ↔ Village C = 5 km

Village B ↔ Village D = 6 km
```

If the government installs **all** the cables,

Instead,

- Every village to receive electricity.
- No unnecessary cable connections.
- Minimum installation cost.

Similar applications include

- Computer Networks
- Telephone Networks
- Railway Networks
- Water Supply Systems
- Road Construction
- Internet Fiber Connections
- PCB (Printed Circuit Board) Design


# 8. Why is MST an Optimization Problem?

An **Optimization Problem** is a problem in which we try to either

- Minimize something, or
- Maximize something.

For MST,

our objective is

```
Minimize

↓

Total Edge Weight
```

- Every vertex must be connected.
- No cycles are allowed.

Since we are minimizing the total cost,

# 9. Why Can MST Be Solved Using a Greedy Algorithm?

A Greedy Algorithm always makes the **best possible choice at the current step**.

For MST,

the best immediate choice is

```
Choose the smallest available edge.
```

Suppose we have

```
Edge

Weight

A-B

7

A-C

2

B-C

5
```

The Greedy Algorithm immediately chooses

```
A-C

Weight = 2
```

because it is the smallest edge.

Then,

it again chooses the next smallest edge that

- does not create a cycle, and
- keeps the graph connected.

This process continues until all vertices are connected.

Thus,

the algorithm repeatedly makes the **locally optimal choice**, hoping to obtain the **globally optimal solution**.

This is exactly the Greedy strategy.

# 10. Why Does Kruskal's Algorithm Work?

Kruskal's Algorithm follows two simple rules.

### Rule 1

Always select the **smallest weighted edge** first.

### Rule 2

Never select an edge that creates a cycle.

These two rules ensure that

- The graph remains connected.
- The total cost remains minimum.
- No unnecessary edges are included.

Because of these rules,

Kruskal's Algorithm always constructs a valid Minimum Spanning Tree.


# 11. Important Observations

While studying Minimum Spanning Trees, remember the following points.

- A graph can have many spanning trees.
- Only one (or sometimes multiple with the same weight) is the Minimum Spanning Tree.
- Every MST is a spanning tree.
- Every spanning tree is **not** necessarily an MST.
- An MST always contains **V − 1 edges**.
- An MST never contains a cycle.
- Kruskal's Algorithm always selects the smallest edge first while avoiding cycles.

# 12. Kruskal's Algorithm

After understanding the concept of a **Minimum Spanning Tree (MST)**, the next question is:

> **How do we actually find the Minimum Spanning Tree?**

There are several algorithms to find an MST, but one of the simplest and most popular algorithms is **Kruskal's Algorithm**.

Kruskal's Algorithm was developed by **Joseph Kruskal** in **1956**.

It follows the **Greedy Algorithm** approach because at every step, it makes the **best possible local decision**, hoping to obtain the globally optimal solution.

Instead of considering the complete graph at once, Kruskal's Algorithm repeatedly selects the **smallest weighted edge** that **does not create a cycle**.

The process continues until all vertices become connected.


# 13. Basic Idea Behind Kruskal's Algorithm

Suppose you have five cities that need to be connected.

There are many possible roads between them.

```
          City A

        /   |    \

       /    |     \

   City B  City C  City D

        \     |

         \    |

          City E
```

Each road has a construction cost.

Your goal is

- Connect every city.
- Spend the least amount of money.
- Avoid constructing unnecessary roads.

Now think like a greedy person.

Instead of analyzing every possible combination,

you simply ask,

> **"Which is the cheapest road available right now?"**

You build that road.

Again,

you ask,

> **"Among the remaining roads, which is the cheapest road that does not create a cycle?"**

You continue this process until all cities become connected.

This is exactly the working principle of Kruskal's Algorithm.

# 20. C++ Implementation of Kruskal's Algorithm

The following C++ program implements **Kruskal's Algorithm** using the **Disjoint Set Union (DSU)** data structure (also called Union-Find).

The DSU data structure helps us efficiently detect whether adding an edge creates a **cycle**.

Every important line of the program is explained using comments so that beginners can understand **why** each statement is written.


## 20.1 C++ Code (Fully Commented)

```cpp
#include <iostream>     // Used for input and output operations
#include <vector>       // Used to store edges dynamically
#include <algorithm>    // Used for sorting the edges

using namespace std;

//------------------------------------------------------------
// Structure to represent an Edge of the graph
//------------------------------------------------------------
struct Edge
{
    int src;       // Starting vertex of the edge
    int dest;      // Ending vertex of the edge
    int weight;    // Weight (or cost) of the edge
};

//------------------------------------------------------------
// Comparator Function
// Used by sort() to arrange edges in ascending order
//------------------------------------------------------------
bool compare(Edge a, Edge b)
{
    return a.weight < b.weight;
}

//------------------------------------------------------------
// Disjoint Set Union (Union-Find)
//------------------------------------------------------------
class DSU
{
private:

    vector<int> parent;     // Stores the parent of each vertex
    vector<int> rank;       // Stores the approximate height of each tree

public:

    //--------------------------------------------------------
    // Constructor
    //--------------------------------------------------------
    DSU(int n)
    {
        parent.resize(n);
        rank.resize(n, 0);

        // Initially every vertex is its own parent.
        for(int i = 0; i < n; i++)
        {
            parent[i] = i;
        }
    }

    //--------------------------------------------------------
    // Find Function
    // Finds the representative(parent) of a set.
    //--------------------------------------------------------
    int find(int x)
    {
        // If x is its own parent,
        // then x is the representative of the set.
        if(parent[x] == x)
            return x;

        // Path Compression
        // Makes future searches much faster.
        return parent[x] = find(parent[x]);
    }

    //--------------------------------------------------------
    // Union Function
    // Combines two different sets.
    //--------------------------------------------------------
    void unite(int x, int y)
    {
        int rootX = find(x);
        int rootY = find(y);

        // Both vertices already belong
        // to the same set.
        // Adding this edge creates a cycle.
        if(rootX == rootY)
            return;

        // Union by Rank
        // Attach the smaller tree
        // below the larger tree.
        if(rank[rootX] < rank[rootY])
        {
            parent[rootX] = rootY;
        }
        else if(rank[rootX] > rank[rootY])
        {
            parent[rootY] = rootX;
        }
        else
        {
            parent[rootY] = rootX;
            rank[rootX]++;
        }
    }
};

//------------------------------------------------------------
// Kruskal's Algorithm
//------------------------------------------------------------
void kruskal(vector<Edge> &edges, int vertices)
{
    // Sort all edges according to weight
    sort(edges.begin(), edges.end(), compare);

    // Create Disjoint Set
    DSU dsu(vertices);

    int totalWeight = 0;

    cout << "Edges in Minimum Spanning Tree\n\n";

    // Traverse every edge
    for(Edge edge : edges)
    {
        // Find the parent of both vertices
        int rootU = dsu.find(edge.src);
        int rootV = dsu.find(edge.dest);

        // If both vertices belong
        // to different components,
        // then adding this edge
        // will NOT create a cycle.
        if(rootU != rootV)
        {
            cout << edge.src
                 << " - "
                 << edge.dest
                 << " : "
                 << edge.weight
                 << endl;

            totalWeight += edge.weight;

            // Merge both components
            dsu.unite(rootU, rootV);
        }
    }

    cout << "\nTotal Weight = "
         << totalWeight
         << endl;
}

//------------------------------------------------------------
// Driver Function
//------------------------------------------------------------
int main()
{
    int vertices = 4;

    vector<Edge> edges =
    {
        {0,1,10},
        {0,2,6},
        {0,3,5},
        {1,3,15},
        {2,3,4}
    };

    kruskal(edges, vertices);

    return 0;
}
```


# 21. Time Complexity Analysis

Let,

```
V = Number of Vertices

E = Number of Edges
```

The algorithm performs three major operations.


## Step 1 : Sorting the Edges

Before selecting the edges,

Kruskal's Algorithm sorts all the edges in increasing order.

```
sort(edges.begin(), edges.end(), compare);
```

Sorting takes

```
O(E log E)
```

time.


## Step 2 : Traversing Every Edge

The algorithm visits every edge exactly once.

```
for(each edge)
```

This requires

```
O(E)
```

iterations.


## Step 3 : Find and Union Operations

For every edge,

the algorithm performs

- Find()
- Find()
- Union() (only if required)

Using

- **Path Compression**
- **Union by Rank**

each operation takes nearly constant time.

Mathematically,

```
O(α(V))
```

where

```
α(V)
```

is the **Inverse Ackermann Function**, which grows extremely slowly and is practically constant for all realistic input sizes.

Therefore,

processing all edges requires

```
O(E × α(V))
```

Since

```
α(V) ≈ 1
```

this becomes approximately

```
O(E)
```


## Overall Time Complexity

The dominant operation is sorting.

Therefore,

```
O(E log E)

+

O(E)

≈

O(E log E)
```

### Final Time Complexity

```
O(E log E)
```


# 22. Space Complexity Analysis

The algorithm stores

- Parent Array
- Rank Array
- Edge List

### Parent Array

Stores one parent for every vertex.

```
O(V)
```


### Rank Array

Stores one rank for every vertex.

```
O(V)
```


### Edge List

Stores all edges.

```
O(E)
```


### Recursion Stack

The recursive `find()` function uses path compression. Due to path compression, the recursion depth is very small in practice and is often treated as negligible compared to the edge storage.

---

## Overall Space Complexity

```
O(V)

+

O(V)

+

O(E)

=

O(V + E)
```

### Final Space Complexity

```
O(V + E)
```


# 23. Summary

| Parameter | Complexity |
|-----------|------------|
| Sorting Edges | **O(E log E)** |
| Find Operation | **O(α(V))** (Nearly Constant) |
| Union Operation | **O(α(V))** (Nearly Constant) |
| Overall Time Complexity | **O(E log E)** |
| Space Complexity | **O(V + E)** |

> **Key Point:** Kruskal's Algorithm is efficient because it first sorts the edges and then uses the **Disjoint Set Union (Union-Find)** data structure with **Path Compression** and **Union by Rank** to detect cycles efficiently. These optimizations make the `find()` and `union()` operations almost constant time in practice.

# 24. Prim's Algorithm

After learning **Kruskal's Algorithm**, let us study another popular algorithm used to find the **Minimum Spanning Tree (MST)** called **Prim's Algorithm**.

Like Kruskal's Algorithm, Prim's Algorithm is also a **Greedy Algorithm**.

However, the working approach of both algorithms is different.

- **Kruskal's Algorithm** focuses on selecting the **smallest edge** from the entire graph.
- **Prim's Algorithm** focuses on **expanding the existing tree** by selecting the minimum weighted edge connected to the current tree.


# 25. Basic Idea Behind Prim's Algorithm

Imagine you are constructing roads to connect different cities.

Initially, you are standing in **one city**.

Instead of looking at **all roads in the graph**, you only look at the roads connected to the city (or cities) that are already connected.

Among those roads, you always choose the **road with the smallest cost**.

After selecting that road, the new city becomes part of your connected network.

You repeat this process until **every city is connected**.

This is exactly how Prim's Algorithm works.


# 26. Steps of Prim's Algorithm

Prim's Algorithm follows these steps:

### Step 1

Choose any vertex as the starting vertex.

```
Example

Start from Vertex A
```


### Step 2

Mark the starting vertex as **visited**.

Current MST

```
A
```


### Step 3

Find all the edges connected to the visited vertices.

Example

```
A → B = 4

A → C = 2

A → D = 8
```


### Step 4

Select the edge having the **minimum weight**.

```
Choose

A → C

Weight = 2
```

Current MST

```
A ----- C
```


### Step 5

Mark the newly added vertex as visited.

Visited vertices

```
A

C
```


### Step 6

Again, consider **all edges connected to the visited vertices**.

Ignore

- Edges leading to already visited vertices.
- Edges creating cycles.

Choose the smallest valid edge.

Repeat until

```
Number of Selected Edges = V - 1
```

# 29. C++ Implementation of Prim's Algorithm

The following program uses an **Adjacency Matrix** to represent the graph.

Each important line is explained using comments.

```cpp
#include <iostream>     // Used for input and output
#include <climits>      // Provides INT_MAX

using namespace std;

#define V 5             // Total number of vertices

//-------------------------------------------------------
// Function to find the vertex having
// minimum key value.
//-------------------------------------------------------
int minKey(int key[], bool mstSet[])
{
    int minimum = INT_MAX;
    int index = -1;

    // Traverse all vertices
    for(int i = 0; i < V; i++)
    {
        // Select the smallest key
        // among the unvisited vertices.
        if(mstSet[i] == false && key[i] < minimum)
        {
            minimum = key[i];
            index = i;
        }
    }

    return index;
}

//-------------------------------------------------------
// Function to print MST
//-------------------------------------------------------
void printMST(int parent[], int graph[V][V])
{
    cout << "Edge\tWeight\n";

    for(int i = 1; i < V; i++)
    {
        cout << parent[i]
             << " - "
             << i
             << "\t"
             << graph[i][parent[i]]
             << endl;
    }
}

//-------------------------------------------------------
// Prim's Algorithm
//-------------------------------------------------------
void primMST(int graph[V][V])
{
    int parent[V];      // Stores the MST

    int key[V];         // Stores minimum edge weight

    bool mstSet[V];     // Keeps track of included vertices

    // Initialize all keys as infinity
    for(int i = 0; i < V; i++)
    {
        key[i] = INT_MAX;
        mstSet[i] = false;
    }

    // Start from vertex 0
    key[0] = 0;

    // Root has no parent
    parent[0] = -1;

    // Repeat V-1 times
    for(int count = 0; count < V - 1; count++)
    {
        // Select minimum key vertex
        int u = minKey(key, mstSet);

        // Include it in MST
        mstSet[u] = true;

        // Update adjacent vertices
        for(int v = 0; v < V; v++)
        {
            // Update key only if
            // edge exists,
            // vertex is not in MST,
            // and edge weight is smaller.
            if(graph[u][v] &&
               mstSet[v] == false &&
               graph[u][v] < key[v])
            {
                parent[v] = u;
                key[v] = graph[u][v];
            }
        }
    }

    printMST(parent, graph);
}

//-------------------------------------------------------
// Driver Function
//-------------------------------------------------------
int main()
{
    int graph[V][V] =
    {
        {0,2,0,6,0},
        {2,0,3,8,5},
        {0,3,0,0,7},
        {6,8,0,0,9},
        {0,5,7,9,0}
    };

    primMST(graph);

    return 0;
}
```

# 30. Time Complexity Analysis

Let,

```
V = Number of Vertices
```

The algorithm performs

```
V-1
```

iterations.

For every iteration,

it searches for the minimum key vertex.

Searching requires

```
O(V)
```

Updating adjacent vertices also requires

```
O(V)
```

Therefore,

```
O(V × V)

=

O(V²)
```

### Final Time Complexity (Adjacency Matrix)

```
O(V²)
```

> **Note:** If Prim's Algorithm is implemented using an **Adjacency List** with a **Min-Heap (Priority Queue)**, the time complexity improves to **O(E log V)**.

> **Key Point:** Prim's Algorithm grows the Minimum Spanning Tree one vertex at a time by always selecting the **minimum-weight edge** that connects a visited vertex to an unvisited vertex. Unlike Kruskal's Algorithm, it does **not** sort all edges beforehand and naturally builds a single connected tree from the chosen starting vertex.
