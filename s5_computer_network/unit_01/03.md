# Network Software and Network Simulators

---

## 1. Network Software

### Definition

Network software refers to all the programs, protocols, and operating system components that control, manage, and enable communication between devices in a computer network.

### Key Points

- Hardware alone cannot make a network functional.
- Network software acts as the brain of the network — it defines how data is prepared, sent, received, and interpreted.
- Network software includes protocol stacks, network operating systems, network management software, and security software.
- Without network software, even if all devices are physically connected, they cannot communicate with each other.

---

### 1.1 Protocol Stack

#### Definition

A protocol stack is a set of communication protocols arranged in layers, where each layer has a specific role in the process of data transmission, and together all layers work as a complete system to enable communication between devices.

#### Key Points

- Also called a network stack or protocol suite.
- Each layer in the stack performs a specific function and communicates only with the layer directly above or below it.
- The most widely used protocol stack is TCP/IP.
- Another important reference model is the OSI (Open Systems Interconnection) model which has 7 layers.
- The concept of layering makes it easier to design, troubleshoot, and upgrade networks because each layer is independent.

#### Why Layering is Used

1. **Simplicity** - Each layer handles only one specific task, making the overall system easier to understand and manage.
2. **Modularity** - Changes in one layer do not affect other layers.
3. **Standardization** - Different manufacturers can build hardware and software that work together because all follow the same layered standards.
4. **Easy Troubleshooting** - If there is a problem, it can be isolated to a specific layer quickly.

---

#### 1.1.1 OSI Model (Open Systems Interconnection Model)

- Definition: The OSI model is a conceptual framework developed by ISO (International Organization for Standardization) that divides network communication into 7 distinct layers, each with a specific function.
- It was introduced in 1984.
- It is a reference model — meaning it is used for understanding and designing networks, not an actual implementation.

| Layer Number | Layer Name | Function |
|---|---|---|
| Layer 7 | Application Layer | Provides network services directly to user applications. Example: HTTP, FTP, SMTP |
| Layer 6 | Presentation Layer | Translates, encrypts, and compresses data. Example: SSL/TLS encryption |
| Layer 5 | Session Layer | Manages sessions (start, maintain, terminate connections) between applications |
| Layer 4 | Transport Layer | Ensures reliable data transfer; handles error correction and flow control. Example: TCP, UDP |
| Layer 3 | Network Layer | Handles logical addressing and routing of data packets. Example: IP, ICMP |
| Layer 2 | Data Link Layer | Handles MAC addressing and error detection in frames. Example: Ethernet, Wi-Fi |
| Layer 1 | Physical Layer | Deals with actual transmission of raw bits over a physical medium. Example: Cables, NIC |

- Memory Tip to remember layers from top to bottom: **All People Seem To Need Data Processing**
  - Application, Presentation, Session, Transport, Network, Data Link, Physical

---

#### 1.1.2 TCP/IP Model

- Definition: The TCP/IP model is a practical, four-layer protocol stack that defines how data is transmitted over the internet and most modern networks.
- It is the actual protocol suite used on the internet.
- Developed by the US Department of Defense in the 1970s.

| Layer Number | Layer Name | Corresponding OSI Layers | Example Protocols |
|---|---|---|---|
| Layer 4 | Application Layer | Application + Presentation + Session | HTTP, FTP, SMTP, DNS |
| Layer 3 | Transport Layer | Transport | TCP, UDP |
| Layer 2 | Internet Layer | Network | IP, ICMP, ARP |
| Layer 1 | Network Access Layer | Data Link + Physical | Ethernet, Wi-Fi |

#### Difference Between OSI Model and TCP/IP Model

| Point | OSI Model | TCP/IP Model |
|---|---|---|
| Number of Layers | 7 Layers | 4 Layers |
| Developed By | ISO | US Department of Defense |
| Type | Reference / Conceptual Model | Practical / Implementation Model |
| Usage | Used for understanding and teaching | Used in actual internet communication |
| Protocol Dependency | Protocol independent | Protocol specific (TCP, IP) |

### 1.2 Key Protocols in the Protocol Stack

#### Definition

A protocol is a set of rules that defines how data is formatted, transmitted, and received between two devices on a network.

---

**1. HTTP (HyperText Transfer Protocol)**
- Used for transferring web pages over the internet.
- When a user types a website URL in a browser, HTTP is used to fetch the web page from the server.
- Example: Accessing www.google.com uses HTTP/HTTPS.

**2. HTTPS (HTTP Secure)**
- A secure version of HTTP that uses SSL/TLS encryption to protect data during transfer.
- Used for secure transactions like online banking and shopping.
- Example: Any website with a padlock icon in the browser uses HTTPS.

**3. FTP (File Transfer Protocol)**
- Used to transfer files from one computer to another over a network.
- Supports both uploading and downloading of files.
- Example: A web developer uploads website files to a web server using FTP.

**4. SMTP (Simple Mail Transfer Protocol)**
- Used to send emails from a sender's mail server to the recipient's mail server.
- Example: When a user sends an email from Gmail, SMTP is used to deliver the email to the recipient's mail server.

**5. POP3 / IMAP (Email Receiving Protocols)**
- POP3 (Post Office Protocol 3): Downloads emails from the server to the local device and deletes them from the server.
- IMAP (Internet Message Access Protocol): Allows emails to be read directly from the server without downloading; emails remain on the server.
- Example: Email clients like Outlook use IMAP or POP3 to receive emails.

**6. DNS (Domain Name System)**
- Translates human-readable domain names into IP addresses that computers use to identify each other.
- Example: When a user types www.youtube.com, DNS translates it to its IP address (e.g., 142.250.77.206) so the browser can connect to the correct server.

**7. DHCP (Dynamic Host Configuration Protocol)**
- Automatically assigns IP addresses to devices when they connect to a network.
- Example: When a mobile phone connects to a Wi-Fi network, DHCP automatically assigns it an IP address without manual configuration.

**8. TCP (Transmission Control Protocol)**
- A connection-oriented protocol that ensures reliable, ordered, and error-checked delivery of data between applications.
- Example: Downloading a file, loading a web page — both use TCP because data must arrive completely and correctly.

**9. UDP (User Datagram Protocol)**
- A connectionless protocol that sends data without establishing a connection and without guaranteeing delivery.
- Faster than TCP but less reliable.
- Example: Video streaming, online gaming, and voice calls use UDP because speed is more important than perfect accuracy.

**10. IP (Internet Protocol)**
- Responsible for logical addressing and routing of data packets from source to destination across networks.
- Two versions: IPv4 (32-bit address, e.g., 192.168.1.1) and IPv6 (128-bit address).
- Example: Every device connected to the internet has an IP address assigned by IP protocol.

### 1.3 Network Operating System (NOS)

#### Definition

A Network Operating System is a specialized operating system that provides the necessary software environment to support networking functions such as file sharing, printer sharing, user management, security, and communication between devices on a network.

#### Key Points

- A NOS runs on a server and manages network resources and services.
- It controls how data is shared and accessed across the network.
- It provides user authentication (login and password verification) and access control.
- A NOS can support multiple users simultaneously.
- It handles network administration tasks such as adding users, setting permissions, and monitoring traffic.

#### Functions of a Network Operating System

1. **File Sharing** - Allows multiple users to access and share files stored on a central server.
2. **Printer Sharing** - Manages shared printers accessible to all users on the network.
3. **User Authentication** - Verifies the identity of users through usernames and passwords before granting access.
4. **Access Control** - Defines permissions — which user can read, write, or delete specific files or resources.
5. **Network Management** - Monitors network performance, manages connected devices, and detects faults.
6. **Security Management** - Provides firewall, encryption, and intrusion detection to protect the network.
7. **Backup and Recovery** - Manages regular data backups to prevent data loss.

#### Examples of Network Operating Systems

1. **Windows Server** (Microsoft) - Widely used in businesses. Example: Windows Server 2019, 2022.
2. **Linux (Ubuntu Server, CentOS, Red Hat)** - Open-source NOS widely used in enterprises and web servers.
3. **Novell NetWare** - One of the earliest NOS, used in the 1980s and 1990s.
4. **UNIX** - Used in large enterprise environments and research institutions.

#### Example

In a college, a Windows Server-based NOS manages all student login accounts, controls access to the college's file server, manages the shared printer in the lab, and monitors network activity — all from a single central server.

### 1.4 Network Management Software

#### Definition

Network management software is a program or set of tools used to monitor, control, analyze, and maintain the performance and security of a computer network.

#### Key Points

- It allows network administrators to view all devices connected to the network.
- It detects faults, bottlenecks, and security threats in real time.
- It generates reports on network usage and performance.
- It can automatically alert administrators when a device fails or when unusual activity is detected.

#### Functions

1. **Network Monitoring** - Continuously checks the status of all devices and connections.
2. **Performance Analysis** - Measures network speed, bandwidth usage, and latency.
3. **Fault Detection** - Identifies and reports hardware or software failures.
4. **Configuration Management** - Manages and updates the configuration of network devices remotely.
5. **Security Monitoring** - Detects unauthorized access attempts and suspicious network activity.

#### Examples

- **SolarWinds Network Performance Monitor**
- **Nagios**
- **Wireshark** (used for analyzing network traffic/packets)
- **PRTG Network Monitor**

## 2. Network Simulators

### Definition

A network simulator is a software tool that allows users to create, configure, and test virtual computer networks without the need for actual physical hardware devices such as routers, switches, and cables.

### Key Points

- Simulators create a virtual environment that mimics the behavior of real network devices and protocols.
- They are used for learning, testing, research, and troubleshooting network configurations.
- Students can practice configuring routers and switches without purchasing expensive hardware.
- Network engineers test new network designs in a simulator before implementing them in a real network to avoid costly mistakes.
- Simulators are widely used in university labs, certification training, and research institutions.

### Advantages of Using Network Simulators

1. **Cost Effective** - No need to purchase expensive physical routers, switches, or cables.
2. **Safe Testing Environment** - New configurations and changes can be tested without affecting a live network.
3. **Learning Tool** - Students can experiment and learn networking concepts practically.
4. **Repeatability** - The same network scenario can be tested multiple times easily.
5. **Accessibility** - Can be run on a personal laptop or computer without any special setup.

---

### 2.1 Cisco Packet Tracer

#### Definition

Cisco Packet Tracer is a network simulation software developed by Cisco Systems that allows users to design, configure, and simulate computer networks using virtual Cisco routers, switches, and end devices.

#### Key Points

- Developed and provided free of cost by Cisco Systems.
- Primarily used for learning and practicing networking concepts, especially for Cisco certifications (CCNA).
- Supports a wide range of network devices including routers, switches, hubs, firewalls, wireless access points, and end devices.
- Users can configure devices using Cisco IOS (Internetwork Operating System) commands — the same commands used on real Cisco hardware.
- Supports simulation of protocols like TCP/IP, OSPF, RIP, EIGRP, VLANs, and more.
- Includes a visual mode where users can see data packets moving across the network in real time.

#### Features

1. **Device Library** - Contains a wide range of virtual Cisco devices that can be dragged and dropped onto the workspace.
2. **CLI Access** - Devices can be configured using the command-line interface, exactly like a real Cisco device.
3. **Simulation Mode** - Allows users to observe how data packets travel through the network step by step.
4. **Activity Wizard** - Teachers can create network lab exercises and assignments for students.
5. **Multi-User Mode** - Multiple students can collaborate on the same network design simultaneously.

#### Who Uses It

- Students preparing for Cisco CCNA, CCNP certifications.
- B.Tech / BCA / MCA students learning computer networks in university labs.
- Network engineers prototyping new network designs.

#### Example

A student can create a network in Packet Tracer with two routers, three switches, and ten computers, configure IP addresses and routing protocols, and then test if the computers can communicate with each other — all without any physical hardware.

---

### 2.2 GNS3 (Graphical Network Simulator 3)

#### Definition

GNS3 is an open-source network simulation software that allows users to emulate real network devices by running actual operating system images of routers, switches, and firewalls on a personal computer.

#### Key Points

- GNS3 stands for Graphical Network Simulator 3.
- Unlike Packet Tracer which only simulates devices, GNS3 emulates them — meaning it runs the actual firmware/OS of real Cisco, Juniper, and other vendor devices.
- Because it uses real device OS images, the behavior is identical to real hardware.
- Supports devices from multiple vendors: Cisco, Juniper, MikroTik, Palo Alto, etc.
- Requires more system resources (RAM and CPU) compared to Packet Tracer because it runs actual OS images.
- Used by professionals and advanced students for real-world network testing.
- Open-source and free to download, but requires users to obtain router OS images separately (from their own licensed hardware).

#### Difference Between Packet Tracer and GNS3

| Point | Cisco Packet Tracer | GNS3 |
|---|---|---|
| Developer | Cisco Systems | Open-source community |
| Type | Simulation (virtual behavior) | Emulation (runs real OS) |
| Device Support | Cisco devices only | Multiple vendors (Cisco, Juniper, etc.) |
| Accuracy | Approximate | Identical to real hardware |
| System Resources | Low | High (needs more RAM and CPU) |
| Cost | Free (with Cisco NetAcad account) | Free (open-source) |
| Best For | Beginners and students | Advanced users and professionals |

#### Example

A network engineer can load the actual Cisco IOS image of a router into GNS3 and test complex routing configurations, firewall rules, and MPLS setups — exactly as they would behave on real physical hardware — before deploying changes on a live production network.

### 2.3 NS2 / NS3 (Network Simulator 2 and 3)

#### Definition

NS2 (Network Simulator version 2) and NS3 (Network Simulator version 3) are open-source, discrete-event network simulators primarily used for academic research and simulation of network protocols and algorithms.

#### Key Points

- NS2 was developed at the University of California, Berkeley and is widely used in academic research.
- NS3 is the newer, improved version of NS2 with better architecture and more features.
- These simulators are primarily used for research — not for configuring actual network devices.
- Users write simulation scripts (in TCL language for NS2, and C++ or Python for NS3) to define network topology, protocols, and traffic.
- They are widely used in universities and research institutions to simulate and study network protocols such as TCP, UDP, routing algorithms, wireless protocols (Wi-Fi, LTE), and more.
- Results are generated as log files and trace files which are then analyzed using tools like NAM (Network Animator) for NS2.

#### Key Differences Between NS2 and NS3

| Point | NS2 | NS3 |
|---|---|---|
| Language | TCL + C++ | C++ and Python |
| Architecture | Older, less modular | Newer, more modular and scalable |
| Visualization | NAM (Network Animator) | NetAnim |
| Usage | Legacy research projects | Current academic research |
| Support | Less active development | Actively maintained |

#### Who Uses NS2 / NS3

- M.Tech and Ph.D. students for thesis and research work.
- Researchers studying new network protocols.
- Universities conducting network research projects.

#### Example

A researcher studying the performance of a new wireless routing protocol can write an NS3 simulation script that creates 50 virtual mobile nodes, defines their movement pattern, assigns the new protocol, and then runs the simulation to measure packet delivery ratio, delay, and throughput — all without any physical hardware or real network.

## Important Definitions for Exam (Quick Revision)

- **Network Software:** All programs, protocols, and OS components that control and manage communication between devices in a network.

- **Protocol Stack:** A layered set of communication protocols where each layer performs a specific function in the process of data transmission.

- **OSI Model:** A 7-layer conceptual framework developed by ISO that standardizes the functions of a network into seven distinct layers from Physical to Application.

- **TCP/IP Model:** A 4-layer practical protocol suite used on the internet, consisting of Network Access, Internet, Transport, and Application layers.

- **Protocol:** A set of rules that defines how data is formatted, transmitted, and received between devices on a network.

- **HTTP:** HyperText Transfer Protocol; used for transferring web pages between a web server and a browser over the internet.

- **FTP:** File Transfer Protocol; used to transfer files between computers over a network.

- **SMTP:** Simple Mail Transfer Protocol; used to send emails from the sender's mail server to the recipient's mail server.

- **DNS:** Domain Name System; translates human-readable domain names (e.g., google.com) into IP addresses used by computers.

- **DHCP:** Dynamic Host Configuration Protocol; automatically assigns IP addresses to devices when they connect to a network.

- **TCP:** Transmission Control Protocol; a connection-oriented protocol that ensures reliable and ordered delivery of data.

- **UDP:** User Datagram Protocol; a connectionless protocol that sends data without guaranteeing delivery; used where speed is prioritized over reliability.

- **Network Operating System (NOS):** A specialized operating system that runs on a server and manages network resources, user accounts, file sharing, and security.

- **Network Simulator:** A software tool that creates a virtual network environment to design, configure, and test network setups without physical hardware.

- **Cisco Packet Tracer:** A network simulation tool developed by Cisco Systems used by students to design and simulate networks using virtual Cisco devices.

- **GNS3:** An open-source network emulator that runs actual router and switch OS images to provide a realistic network testing environment for professionals.

- **NS3:** An open-source discrete-event network simulator written in C++ and Python, used primarily for academic research on network protocols and algorithms.

- **Emulation vs Simulation:** Simulation mimics the behavior of a device approximately; emulation runs the actual OS of the device and behaves exactly like real hardware.
