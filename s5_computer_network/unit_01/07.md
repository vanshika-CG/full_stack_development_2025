# Peer-to-Peer Network and Client-Server Network

---

## 1. Peer-to-Peer Network (P2P)

### Definition

A Peer-to-Peer network is a type of network in which all connected computers (called peers) are equal in terms of roles and responsibilities, where each computer can act as both a client (requesting services) and a server (providing services) simultaneously, without any dedicated central server.

### Key Points

- In a P2P network, every computer has equal status — no single computer is in charge.
- Each peer can share its own resources (files, printers, internet connection) directly with other peers.
- There is no dedicated server — each computer manages its own resources and security.
- P2P networks are easy and inexpensive to set up because no special server hardware or software is required.
- P2P networks are best suited for small networks with a limited number of users (typically up to 10 to 15 computers).
- Each user is responsible for managing their own computer's security and shared resources.
- P2P is also widely used on the internet for file sharing applications.

---

### 1.1 How a P2P Network Works

- Each computer in the network is connected to the others directly.
- When Computer A needs a file, it directly requests it from Computer B or any other peer that has the file.
- Computer B responds and sends the file directly to Computer A.
- No third computer or server is involved in this transaction.
- The same Computer A that requested a file can also respond to a request from Computer C for another file — acting as a server at the same time.

---

### 1.2 Types of P2P Networks

**1. Pure P2P Network**
- Definition: A network in which all peers are completely equal and there is absolutely no central server or coordinator of any kind.
- Every peer discovers other peers and shares resources entirely on its own.
- Example: Early versions of Gnutella file-sharing network.

**2. Hybrid P2P Network**
- Definition: A network that combines P2P and client-server models — a central server is used only to help peers find each other, but the actual data transfer happens directly between peers.
- The central server acts as a directory or index, not as a data server.
- Example: BitTorrent uses a central tracker server to help peers locate each other, but files are transferred directly between peers.

---

### 1.3 Advantages of P2P Network

1. **Low Cost** - No need to purchase dedicated server hardware or server operating system licenses, which significantly reduces setup cost.
2. **Easy to Set Up** - Simple to configure and does not require professional network administration.
3. **No Single Point of Failure** - Since there is no central server, the failure of one computer does not bring down the entire network. Other computers can continue to function.
4. **Decentralized** - Resources are distributed across all peers, so no single computer bears the entire load.
5. **Scalable for File Sharing** - In internet-based P2P, as more users join and share files, the network becomes more efficient because there are more sources to download from.
6. **Direct Communication** - Peers communicate directly without going through a central server, reducing latency.

---

### 1.4 Disadvantages of P2P Network

1. **Security Risk** - Since each computer manages its own security, it is difficult to enforce a uniform security policy across the network. Viruses and malicious files can spread easily.
2. **Difficult to Manage** - In a large P2P network, managing all computers individually is very difficult because there is no central point of administration.
3. **No Centralized Backup** - There is no central backup system. Each user is responsible for their own data backup, which can lead to data loss.
4. **Performance Depends on Individual Computers** - If a peer sharing a resource has a slow computer or slow internet connection, the performance suffers for the requesting peer.
5. **Not Suitable for Large Networks** - P2P becomes very difficult to manage and inefficient as the number of computers increases beyond 10 to 15.
6. **Data Inconsistency** - Since data is stored on multiple individual computers, there may be multiple different versions of the same file on different peers.
7. **No Centralized Access Control** - It is difficult to restrict which users can access which resources because there is no central authentication system.

---

### 1.5 Examples of P2P Networks

1. **BitTorrent** - A widely used P2P protocol for sharing large files. Users download and upload pieces of a file simultaneously from and to multiple peers.
2. **Gnutella** - One of the earliest P2P file-sharing networks where users shared music and files directly.
3. **Skype (Early Version)** - Originally used a P2P architecture to route voice calls directly between users.
4. **Home Network** - A small home network where two or three computers share files and a printer directly with each other without a dedicated server.
5. **Blockchain Networks** - Bitcoin and other blockchain networks operate on a P2P model where each node has a copy of the entire ledger.

## 2. Client-Server Network

### Definition

A Client-Server network is a type of network in which one or more powerful computers (called servers) provide services, data, and resources to other computers (called clients) that request those services, with a clear distinction between the role of the server and the role of the client.

### Key Points

- In a client-server network, roles are clearly divided:
  - **Server** - Provides services and resources. Runs continuously. Examples: web server, file server, database server.
  - **Client** - Requests and uses services provided by the server. Examples: a user's laptop, desktop, or smartphone.
- The server is a dedicated, powerful machine with high processing power, large storage, and a server operating system.
- A single server can serve multiple clients simultaneously.
- All resources, data, and security are managed centrally on the server.
- Clients do not share resources with each other directly — all sharing happens through the server.
- Client-server is the dominant model used in businesses, organizations, and on the internet.

---

### 2.1 How a Client-Server Network Works

- The server runs continuously and waits for client requests.
- A client sends a request to the server asking for a specific service or resource.
- The server processes the request, retrieves the required data or performs the required action.
- The server sends the response back to the client.
- The client receives the response and displays or uses the result.

**Example Flow:**
1. A student (client) opens a browser and types www.google.com.
2. The browser sends an HTTP request to Google's web server.
3. Google's web server receives the request, processes it, and retrieves the Google homepage.
4. The server sends the HTML, CSS, and JavaScript files back to the student's browser.
5. The browser displays the Google homepage to the student.

---

### 2.2 Types of Client-Server Architecture

**1. Two-Tier Architecture**
- Definition: The client communicates directly with the server. The application logic is split between the client and the server.
- The client handles the user interface and some application logic.
- The server handles the database.
- Simple and fast but not scalable for large applications.
- Example: A desktop application that connects directly to a database server.

**2. Three-Tier Architecture**
- Definition: The system is divided into three separate layers — Presentation Layer (client), Application Layer (application server), and Data Layer (database server).
- The client only handles the user interface.
- The application server handles all business logic and processing.
- The database server stores and manages data.
- More scalable and secure than two-tier.
- Example: A web application where the browser is the client, a web server handles logic, and a separate database server stores data.

**3. N-Tier Architecture**
- Definition: An extension of three-tier architecture with multiple specialized layers between the client and the database server.
- Used in large enterprise and cloud-based applications.
- Example: Large e-commerce platforms like Amazon that have separate layers for user interface, business logic, payment processing, inventory management, and data storage.

---

### 2.3 Advantages of Client-Server Network

1. **Centralized Management** - All data, resources, and user accounts are managed from one central server, making administration easy and efficient.
2. **High Security** - Security policies, user authentication, and access control are enforced centrally on the server, making it easier to maintain a secure environment.
3. **Centralized Backup** - All data is stored on the server, so regular backups can be taken from one central location, reducing the risk of data loss.
4. **Scalability** - New clients can be added to the network easily without major changes to the server or network infrastructure.
5. **Resource Sharing** - Expensive resources like high-capacity storage, printers, and specialized software are shared efficiently among all clients through the server.
6. **Better Performance** - Servers are powerful machines designed to handle many requests simultaneously, providing fast and reliable service to all clients.
7. **Data Consistency** - Since all data is stored centrally on the server, all clients access the same, most up-to-date version of the data.

---

### 2.4 Disadvantages of Client-Server Network

1. **Single Point of Failure** - If the central server fails or goes down, all clients lose access to resources and services until the server is restored.
2. **High Cost** - Setting up a client-server network requires investment in a powerful dedicated server, server operating system licenses, and professional network administration.
3. **Requires Professional Administration** - A trained network administrator is needed to manage, maintain, and secure the server and the network.
4. **Server Overload** - If too many clients send requests simultaneously, the server can become overloaded and slow down, affecting all clients.
5. **Dependency on Server** - Clients cannot access shared resources if the server is unavailable, even if the network itself is functioning.

---

### 2.5 Examples of Client-Server Networks

1. **Web Browsing** - A browser (client) sends requests to a web server (server) which responds with web pages.
2. **Email** - An email client (Gmail app) sends and receives emails through a mail server.
3. **Online Banking** - A user's mobile app (client) communicates with the bank's application server and database server.
4. **College Network** - Student computers (clients) access files, software, and the internet through a central college server.
5. **Online Gaming** - Players' devices (clients) connect to a central game server that manages the game state and synchronizes all players.

## 3. Comparison: P2P vs Client-Server Network

### 3.1 Difference Table

| Feature | Peer-to-Peer (P2P) | Client-Server |
|---|---|---|
| Role of Computers | Every computer acts as both client and server | Roles are fixed — dedicated servers and dedicated clients |
| Central Server | No dedicated central server | Has one or more dedicated central servers |
| Cost | Low — no special server hardware needed | High — requires powerful dedicated server hardware |
| Security | Low — each computer manages its own security | High — security managed centrally on the server |
| Management | Difficult — each computer managed individually | Easy — centralized administration from the server |
| Data Storage | Data stored on individual peer computers | Data stored centrally on the server |
| Data Consistency | Low — different versions may exist on different peers | High — all clients access the same data on the server |
| Backup | No centralized backup | Centralized backup from the server |
| Scalability | Suitable for small networks (up to 10–15 computers) | Highly scalable — can support hundreds or thousands of clients |
| Single Point of Failure | No — failure of one peer does not affect others | Yes — if server fails, all clients lose access |
| Performance | Depends on individual peer's hardware | High — dedicated powerful server handles all requests |
| Setup Complexity | Simple and easy | Complex — requires professional setup |
| Administration | No dedicated administrator needed | Requires trained network administrator |
| Suitable For | Small home networks, file sharing applications | Businesses, organizations, internet services |
| Example | BitTorrent, Home network | Website hosting, College network, Online banking |

---

### 3.2 When to Use P2P and When to Use Client-Server

**Use P2P when:**
- The network has a small number of users (less than 15).
- Budget is limited and a dedicated server is not affordable.
- The primary need is simple file or printer sharing between a few computers.
- No professional network administrator is available.
- Example: A small home setup with 2 to 3 computers sharing files and a printer.

**Use Client-Server when:**
- The network has a large number of users.
- High security and centralized data management are required.
- The organization needs centralized backup and access control.
- Professional administration is available.
- Example: A company with 200 employees, a hospital with patient records, a university network.

## Important Definitions for Exam (Quick Revision)

- **Peer-to-Peer (P2P) Network:** A type of network in which all connected computers are equal in role, where each computer can act as both a client and a server simultaneously, without any dedicated central server.

- **Client-Server Network:** A type of network in which dedicated server computers provide services and resources to client computers that request those services, with clearly defined and fixed roles for servers and clients.

- **Client:** A computer or device that requests and uses services or resources provided by a server over a network.

- **Server:** A powerful, dedicated computer that provides services, data, or resources to client computers connected to it through a network.

- **Peer:** A computer in a P2P network that has equal status with all other computers and can act as both a client and a server.

- **Two-Tier Architecture:** A client-server model in which the client communicates directly with the database server, with application logic split between the two.

- **Three-Tier Architecture:** A client-server model divided into three layers — Presentation (client), Application (application server), and Data (database server) — for better scalability and security.

- **Dedicated Server:** A server that is used exclusively for providing network services and is not used as a regular workstation.

- **Centralized Management:** The administration of all network resources, user accounts, and security from a single central server instead of managing each computer individually.

- **Single Point of Failure:** A component in a network whose failure causes the entire network or service to stop functioning. In a client-server network, the server is a single point of failure.

- **Decentralized Network:** A network in which resources and control are distributed across multiple nodes rather than being managed by a single central server. P2P networks are decentralized.

- **BitTorrent:** A widely used hybrid P2P protocol for sharing large files over the internet, where a central tracker helps peers find each other and files are transferred directly between peers.

- **Pure P2P:** A P2P network model in which all peers are completely equal with no central server or coordinator of any kind.

- **Hybrid P2P:** A P2P network model that uses a central server only as a directory to help peers locate each other, while actual data transfer happens directly between peers.

- **N-Tier Architecture:** An extension of three-tier client-server architecture with multiple specialized layers, used in large enterprise and cloud-based applications.
