# Set
 
### What is a Set
 
A set is an unordered collection of unique elements. No two elements in a set can be equal. Sets are mutable — you can add or remove elements — but the elements themselves must be immutable (hashable), just like dictionary keys.
 
Sets are based on the same hash table structure as dictionaries. This gives sets two powerful properties: membership testing in O(1) time (regardless of set size), and automatic removal of duplicates.
 
```python
# Creating sets
fruits = {"apple", "banana", "cherry"}
numbers = {1, 2, 3, 4, 5}
 
# Creating an empty set — MUST use set(), not {}
empty_set = set()    # correct
empty_dict = {}      # this creates an empty dictionary, not a set
 
# Creating a set from a list — duplicates are removed automatically
words = ["the", "cat", "sat", "the", "cat", "on", "the", "mat"]
vocabulary = set(words)    # {"the", "cat", "sat", "on", "mat"}
 
print(type({}))        # <class 'dict'>
print(type(set()))     # <class 'set'>
```
 
### Key Properties of Sets
 
**No guaranteed order.** Sets do not maintain insertion order. When you print a set, the order of elements may differ from how you inserted them. You cannot rely on order in a set.
 
**No indexing.** Because sets are unordered, there is no concept of "the element at index 2". You cannot subscript a set.
 
**Unique elements.** If you try to add a duplicate, the set silently ignores it. No error is raised.
 
**Elements must be immutable.** This is the same rule as dictionary keys. Integers, floats, strings, and tuples can be elements. Lists, dicts, and other sets cannot.
 
```python
s = {3, 1, 4, 1, 5, 9, 2, 6}
print(s)       # {1, 2, 3, 4, 5, 6, 9} — duplicates removed, order may vary
 
s[0]           # TypeError: 'set' object is not subscriptable
 
valid_set = {1, "hello", (1, 2)}   # valid — int, str, tuple are all hashable
invalid = {1, [1, 2]}              # TypeError — list is not hashable
```
 
### Why Sets Matter in ML and NLP
 
The core use of sets in ML is vocabulary management and membership testing. When building an NLP pipeline:
- You extract all unique words from a corpus to build a vocabulary (set)
- You check whether a new word is in the vocabulary (membership test)
- You find words common to two documents (intersection)
- You find out-of-vocabulary words in test data (difference)
For large vocabularies with tens of thousands of words, using a set for membership testing instead of a list gives a massive speed advantage.
 
### Set Methods
 
```python
s = {1, 2, 3, 4, 5}
 
# add() — adds a single element — no error if already present
s.add(6)      # {1, 2, 3, 4, 5, 6}
s.add(3)      # does nothing — 3 already exists
 
# update() — adds multiple elements from an iterable
s.update([7, 8, 9])       # adds 7, 8, 9
s.update("ab")            # adds 'a' and 'b' as individual characters
s.update([1, 2], [3, 4])  # can take multiple iterables
 
# remove() — removes an element — raises KeyError if not present
s.remove(3)    # removes 3
 
# discard() — removes an element — no error if not present (safer)
s.discard(3)   # removes 3 if present, otherwise does nothing
s.discard(99)  # no error even though 99 is not in s
 
# pop() — removes and returns an ARBITRARY element — sets are unordered
element = s.pop()   # you cannot predict which element gets removed
 
# clear() — empties the set
s.clear()      # set()
 
# copy() — creates a shallow copy
s2 = s.copy()
```
 
### Set Operations
 
This is the most powerful feature of sets. Python sets support all standard mathematical set operations directly with operators or methods.
 
```python
A = {1, 2, 3, 4, 5}
B = {4, 5, 6, 7, 8}
 
# Union — all elements from both sets (duplicates removed automatically)
A | B              # {1, 2, 3, 4, 5, 6, 7, 8}
A.union(B)         # same result — method form
 
# Intersection — only elements present in BOTH sets
A & B              # {4, 5}
A.intersection(B)  # same result
 
# Difference — elements in A that are NOT in B
A - B              # {1, 2, 3}
A.difference(B)    # same result
 
# Difference is NOT symmetric — order matters
B - A              # {6, 7, 8}  — elements in B not in A
 
# Symmetric Difference — elements in either A or B but NOT in both
A ^ B              # {1, 2, 3, 6, 7, 8}
A.symmetric_difference(B)    # same result
 
# Subset — is every element of A inside B?
{1, 2}.issubset({1, 2, 3, 4})       # True
{1, 2, 3, 4}.issubset({1, 2})       # False
 
# Superset — does A contain all elements of B?
{1, 2, 3, 4}.issuperset({1, 2})     # True
{1, 2}.issuperset({1, 2, 3, 4})     # False
 
# Disjoint — do A and B share no elements at all?
{1, 2}.isdisjoint({3, 4})    # True  — no common elements
{1, 2}.isdisjoint({2, 3})    # False — 2 is common
```
 
### Membership Testing — O(1) vs O(n)
 
This is where sets dramatically outperform lists. A list must scan every element one by one to check membership — O(n). A set computes the hash of the element and jumps directly to the correct location — O(1).
 
For a vocabulary of 50,000 words, a set lookup is roughly 50,000 times faster than a list lookup.
 
```python
# With a set — O(1) — constant time no matter the size
vocab_set = {"the", "cat", "sat", "on", "mat"}
"the" in vocab_set       # True  — instant
"dog" in vocab_set       # False — instant
"dog" not in vocab_set   # True
 
# With a list — O(n) — gets slower as list grows
vocab_list = ["the", "cat", "sat", "on", "mat"]
"the" in vocab_list      # True  — scans from the beginning
 
# Performance difference becomes critical at scale
# If vocab has 100,000 words and you check 1 million tokens,
# a set will be orders of magnitude faster than a list.
```
 
### Frozenset
 
A frozenset is an immutable version of a set. Once created, elements cannot be added or removed. Because it is immutable, a frozenset is hashable and can be used as a dictionary key or stored inside another set.
 
```python
fs = frozenset([1, 2, 3])
fs.add(4)       # AttributeError — frozenset has no add method
 
# Can be used as a dictionary key
data = {frozenset([1, 2]): "combination A", frozenset([3, 4]): "combination B"}
 
# Can be stored inside a set
set_of_frozensets = {frozenset([1, 2]), frozenset([3, 4])}
```
 
### Common Set Patterns in ML and NLP
 
```python
# Removing duplicate tokens while keeping a list
tokens = ["the", "cat", "sat", "the", "cat"]
unique_tokens = list(set(tokens))    # ["the", "cat", "sat"] — order not guaranteed
 
# Building vocabulary from a corpus
corpus = ["the cat sat", "the dog ran", "cat and dog"]
vocab = set()
for sentence in corpus:
    vocab.update(sentence.split())
# vocab = {"the", "cat", "sat", "dog", "ran", "and"}
 
# Finding common words between two documents
doc1_words = set("machine learning is great".split())
doc2_words = set("deep learning is powerful".split())
 
common_words = doc1_words & doc2_words       # {"learning", "is"}
only_in_doc1 = doc1_words - doc2_words       # {"machine", "great"}
all_words = doc1_words | doc2_words          # all unique words across both
 
# Finding out-of-vocabulary (OOV) words
train_vocab = {"the", "cat", "sat", "on", "mat"}
test_sentence = {"the", "cat", "flew", "over", "mat"}
oov_words = test_sentence - train_vocab      # {"flew", "over"}
 
# Validating prediction labels
valid_classes = {"cat", "dog", "bird"}
predicted_label = "fish"
if predicted_label not in valid_classes:
    print(f"Invalid prediction: {predicted_label}")
 
# Jaccard similarity between two documents
def jaccard_similarity(doc1_words, doc2_words):
    s1 = set(doc1_words.split())
    s2 = set(doc2_words.split())
    intersection = s1 & s2
    union = s1 | s2
    return len(intersection) / len(union) if union else 0.0
```
