# Strings

 
### What is a String
 
A string is a sequence of characters. In Python, a string is an object of type `str`. Every character in a string — letters, digits, spaces, punctuation, newlines — is stored as an individual element in that sequence.
 
Python strings are enclosed in single quotes, double quotes, or triple quotes. All three styles create the same type of object. The choice between them is mostly about convenience.
 
```python
name = "David"
greeting = 'Hello, World'
paragraph = """This is a
multi-line string."""
```
 
Triple quotes are used when the string itself spans multiple lines, or when the string contains both single and double quote characters inside it.
 
### How Strings are Stored Internally
 
Python stores a string as an array of Unicode characters. Each character occupies a position called an index. The first character is at index 0, the second at index 1, and so on. Python also supports negative indexing, where -1 refers to the last character, -2 to the second last, and so on.
 
This is what the string `"Python"` looks like internally:
 
```
P  y  t  h  o  n
0  1  2  3  4  5   (positive index)
-6 -5 -4 -3 -2 -1  (negative index)
```
 
### Why Strings Matter in ML and AI
 
In Natural Language Processing (NLP) and Generative AI, text data is the primary input. Everything — documents, queries, prompts, model outputs — starts as a string. Before feeding text to any ML model, you must preprocess it: clean it, normalize it, split it into tokens, remove unwanted characters. All of this is done through string operations.
 
When working with LLM APIs like OpenAI or Anthropic, the system prompt, user message, and model response are all strings. Understanding how to manipulate strings efficiently is therefore a fundamental skill.
 
### String Immutability
 
Strings in Python are immutable. This means once a string is created, you cannot change any character inside it. Any operation that appears to "modify" a string actually creates a new string and returns it.
 
```python
s = "hello"
s[0] = "H"    # TypeError: 'str' object does not support item assignment
 
# Correct way — create a new string
s = "H" + s[1:]   # "Hello"
```
 
This immutability is intentional. It makes strings safe to use as dictionary keys, allows Python to optimize memory by reusing identical strings, and prevents accidental data corruption.
 
### Indexing and Slicing
 
Indexing retrieves a single character. Slicing retrieves a substring using the syntax `s[start:stop:step]`. The stop index is always excluded.
 
```python
s = "Python"
 
# Indexing
s[0]      # 'P'
s[1]      # 'y'
s[-1]     # 'n'  — last character
s[-2]     # 'o'  — second last
 
# Slicing — s[start:stop:step]
s[0:3]    # 'Pyt'    — index 0, 1, 2. stop=3 is excluded
s[2:]     # 'thon'   — from index 2 to end
s[:4]     # 'Pyth'   — from start to index 3
s[::2]    # 'Pto'    — every second character
s[::-1]   # 'nohtyP' — step=-1 means go backwards, reversing the string
```
 
When any of the three values in slicing are omitted, Python uses defaults. The default start is 0, default stop is the end of the string, and default step is 1.
 
### String Methods
 
String methods are built-in functions that belong to the string object. They always return a new string — they never modify the original because strings are immutable. You call them using dot notation: `string.method()`.
 
```python
text = "  Hello World, this is Python  "
 
# Case methods
# lower() converts every character to lowercase
text.lower()        # "  hello world, this is python  "
 
# upper() converts every character to uppercase
text.upper()        # "  HELLO WORLD, THIS IS PYTHON  "
 
# title() capitalizes the first letter of every word
text.title()        # "  Hello World, This Is Python  "
 
# swapcase() flips lowercase to uppercase and uppercase to lowercase
text.swapcase()     # "  hELLO wORLD, THIS IS pYTHON  "
 
# capitalize() uppercases the very first character and lowercases the rest
# Important: if the first character is a space or digit, no letter gets capitalized
text.capitalize()   # "  hello world, this is python  "
                    # first char is a space, so nothing is capitalized — rest becomes lowercase
 
# Correct examples of capitalize()
"hello world".capitalize()    # "Hello world"
"HELLO WORLD".capitalize()    # "Hello world"  — rest is forced lowercase
"  hello".capitalize()        # "  hello"      — first char is space, unchanged
```
 
```python
text = "  Hello World, this is Python  "
 
# Whitespace methods
# strip() removes whitespace from both the left and right ends
text.strip()        # "Hello World, this is Python"
 
# lstrip() removes whitespace from the left end only
text.lstrip()       # "Hello World, this is Python  "
 
# rstrip() removes whitespace from the right end only
text.rstrip()       # "  Hello World, this is Python"
 
# You can also pass a specific character to strip
"###hello###".strip("#")    # "hello"
```
 
```python
text = "  Hello World, this is Python  "
 
# Search methods
# find() returns the starting index of the first match, or -1 if not found
text.find("World")      # returns 8 (approximately — depends on leading spaces)
text.find("xyz")        # -1
 
# index() does the same as find() but raises ValueError if not found
text.index("World")     # returns index
text.index("xyz")       # ValueError — use find() when unsure if substring exists
 
# count() counts non-overlapping occurrences of a substring
text.count("l")         # 3
text.count("is")        # 2
 
# startswith() and endswith() check the beginning and end of a string
text.startswith("  Hello")   # True
text.endswith("  ")          # True
```
 
```python
text = "  Hello World, this is Python  "
 
# replace() creates a new string with all occurrences replaced
# replace(old, new) — replaces all
text.replace("Python", "ML")     # "  Hello World, this is ML  "
 
# replace(old, new, count) — replaces only the first 'count' occurrences
"aaa".replace("a", "b", 2)       # "bba" — only first 2 replaced
```
 
```python
# split() breaks a string into a list of substrings
sentence = "machine learning is powerful"
 
sentence.split()           # ['machine', 'learning', 'is', 'powerful']
                           # splits on any whitespace by default
 
sentence.split(" ")        # same result here
"a,b,c".split(",")         # ['a', 'b', 'c']
"a,,b".split(",")          # ['a', '', 'b'] — empty string between commas
 
# split(sep, maxsplit) — splits at most maxsplit times
"a,b,c,d".split(",", 2)   # ['a', 'b', 'c,d']
 
# join() is the reverse of split — combines a list of strings into one string
# The string you call join() on becomes the separator
words = ['machine', 'learning', 'is', 'powerful']
" ".join(words)     # 'machine learning is powerful'
",".join(words)     # 'machine,learning,is,powerful'
"".join(words)      # 'machinelearningispowerful'
" | ".join(words)   # 'machine | learning | is | powerful'
```
 
```python
# Check methods — all return True or False
"hello".isalpha()        # True  — all characters are alphabetic
"hello123".isalpha()     # False — contains digits
"12345".isdigit()        # True  — all characters are digits
"12.5".isdigit()         # False — dot is not a digit
"hello".islower()        # True  — all cased characters are lowercase
"HELLO".isupper()        # True  — all cased characters are uppercase
"  ".isspace()           # True  — all characters are whitespace
"Hello World".istitle()  # True  — each word starts with uppercase
"hello123".isalnum()     # True  — all characters are alphabetic or digits
```
 
```python
# Padding and alignment — used in formatting log outputs
"hello".center(11)         # "   hello   "
"hello".center(11, "-")    # "---hello---"
"hello".ljust(10, "-")     # "hello-----"  (left justified, padded right)
"hello".rjust(10, "-")     # "-----hello"  (right justified, padded left)
"42".zfill(5)              # "00042"  — fills with zeros from the left
```
 
### String Formatting
 
String formatting means embedding variable values inside a string. Python has three ways to do this. The f-string is the modern and preferred way.
 
```python
name = "David"
accuracy = 0.9712
epoch = 10
loss = 0.3245
 
# f-strings — Python 3.6+. Prefix the string with f and put variable inside {}
print(f"Name: {name}")
print(f"Accuracy: {accuracy:.2f}")    # 2 decimal places → 0.97
print(f"Accuracy: {accuracy:.4f}")    # 4 decimal places → 0.9712
print(f"Accuracy: {accuracy:.2%}")    # as percentage    → 97.12%
print(f"Epoch {epoch:03d}")           # 3 digits with zero padding → 010
print(f"Loss: {loss:>10.4f}")         # right-aligned in 10 chars
 
# .format() method — Python 3.0+
print("Name: {}".format(name))
print("Epoch: {0}, Loss: {1:.3f}".format(epoch, loss))
print("Name: {name}, Accuracy: {acc:.2f}".format(name=name, acc=accuracy))
 
# % formatting — older C-style, still found in legacy ML code
print("Loss: %.4f" % loss)
print("Epoch: %d, Accuracy: %.2f" % (epoch, accuracy))
```
 
### Common String Operations in ML and NLP
 
```python
# Membership check — is a substring inside a string?
"the" in "the quick brown fox"       # True
"cat" not in "the quick brown fox"   # True
 
# Length
len("Hello World")    # 11 — includes the space
len("")               # 0
 
# Concatenation — joining two strings with +
first = "Hello"
second = " World"
result = first + second    # "Hello World"
# Note: for large amounts of string joining, use "".join() — it is much faster
 
# Repetition
"ha" * 3     # "hahaha"
"-" * 50     # prints a line of 50 dashes — used for formatting output
 
# Multi-line strings for LLM system prompts
system_prompt = """
You are a helpful AI assistant.
Answer only based on the provided context.
Do not hallucinate or make up information.
"""
 
# Converting string to a list of characters
chars = list("Python")    # ['P', 'y', 't', 'h', 'o', 'n']
"".join(chars)            # 'Python' — convert back
 
# Splitting text into sentences
text = "ML is great. DL is powerful. AI is the future."
sentences = text.split(". ")    # ['ML is great', 'DL is powerful', 'AI is the future.']
```
