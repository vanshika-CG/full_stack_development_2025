## 1. Introduction to Python

### What is Python?

**Python** is a high-level, interpreted, general-purpose programming language created by Guido van Rossum and released in 1991.

It is known for its simple, readable syntax and is widely used in web development, automation, data science, and artificial intelligence.

### Key Characteristics

| Feature | Meaning |
|---|---|
| Interpreted | Code runs line by line through the Python interpreter; no separate compilation step is needed before execution. |
| Dynamically typed | A variable's type is decided automatically at runtime, not declared in advance. |
| High-level | Hides low-level details such as memory management from the programmer. |
| Object-oriented | Supports classes and objects (explored in Lecture 5). |
| Cross-platform | The same code runs on Windows, macOS, and Linux without changes. |
| Open source | Free to use, with a large community and ecosystem of libraries. |

### Why Python is Popular

- Simple, English-like syntax that reduces the learning curve.
- A huge standard library plus third-party packages available via `pip`.
- Strong presence in web development through frameworks like **Django** and **Flask**.
- Industry-relevant: this course builds directly toward Django, so a solid grip on these basics matters later.

## 2. Writing and Running a Python Program

Python programs are plain text files saved with a `.py` extension.

```python
# This is a comment - ignored by the interpreter
print("Hello, World!")
```

- `# This is a comment` — anything after `#` on a line is ignored by Python. Comments document code for humans, not the machine.
- `print(...)` — a built-in function that displays output on the screen.
- `"Hello, World!"` — a string literal, passed as an argument to `print()`.

### Running the Program

```bash
python hello.py
```

On systems where Python 2 and Python 3 are both installed, use:

```bash
python3 hello.py
```

> **Important — Interview Tip**
>
> Interviewers sometimes ask why a script fails with `python` but works with `python3`. The answer: many Linux/macOS systems keep `python` pointing to Python 2 (or unbound) for backward compatibility, while `python3` explicitly targets Python 3. Always check with `python --version` before running scripts.

## 3. Indentation: Python's Block Structure

### Definition

**Indentation** is the use of consistent leading whitespace (spaces or tabs) to define the boundaries of a code block, such as the body of a loop, function, or conditional.

Python does **not** use curly braces `{}` like C, C++, or Java. Indentation itself defines structure.

### Correct Example

```python
if 5 > 2:
    print("Five is greater than two")
```

### Incorrect Example

```python
if 5 > 2:
print("Five is greater than two")
```

Running the incorrect version raises:

```text
IndentationError: expected an indented block
```

1. Use a consistent number of spaces for each indentation level — 4 spaces is the standard convention.
2. Never mix tabs and spaces within the same block.
3. A colon `:` at the end of a line (after `if`, `for`, `while`, `def`, `class`, etc.) always signals that an indented block follows.

## 4. Statements in Python

A **statement** is a single instruction that Python executes.

```python
x = 10
print(x)
```

### Multiple Statements on One Line

This is valid using a semicolon `;`, though it is discouraged for readability.

```python
x = 5; y = 10; print(x + y)
```

### One Statement Across Multiple Lines

A backslash `\` allows a single logical line to continue onto the next physical line.

```python
total = 1 + 2 + 3 + \
        4 + 5
```

## 5. Variables

### Definition

A **variable** is a named location in memory used to store a value that can be read or changed later in a program.

Python does not require a variable's type to be declared ahead of time — the type is inferred automatically from the assigned value.

```python
name = "Alice"
age = 21
height = 5.6
```

Here:

- `name` stores a string.
- `age` stores an integer.
- `height` stores a floating-point number.

### Variable Naming Rules

| Rule | Valid Example | Invalid Example |
|---|---|---|
| Must start with a letter or underscore | `_value`, `name1` | `1value` |
| May contain letters, digits, underscores | `student_name` | `student-name` |
| Case-sensitive | `Age` and `age` are different | — |
| Cannot be a Python keyword | `total`, `count` | `for`, `class`, `if` |
| No spaces allowed | `first_name` | `first name` |

### Naming Convention

Python follows the **PEP 8** style guide, which recommends `snake_case` for variable names.

```python
student_name = "John"     # recommended
StudentName = "John"      # avoid — typically reserved for class names
```

> **Important — Interview Tip**
>
> "What is PEP 8?" is a frequently asked question. **PEP 8** is Python's official style guide, defining conventions for naming, indentation, and code layout to keep Python code consistent and readable across projects.

### Assigning Values

```python
# Single assignment
x = 10

# Multiple assignment - same value to multiple variables
a = b = c = 100

# Multiple assignment - different values in one line
x, y, z = 5, 10, 15
print(x, y, z)   # Output: 5 10 15
```

### Variables as References

A Python variable does not store data directly — it stores a **reference** to an object in memory.

```python
x = 5
y = x      # y now refers to the same value as x
x = 10     # reassigning x does not affect y
print(y)   # Output: 5
```

### Checking a Variable's Type

The built-in `type()` function returns the data type of any value or variable.

```python
age = 21
print(type(age))    # <class 'int'>
```

## 6. Data Types in Python

### Definition

A **data type** defines the kind of value a variable can hold and the operations that can be performed on it.

### Overview of Built-in Data Types

| Category | Types |
|---|---|
| Numeric | `int`, `float`, `complex` |
| Text | `str` |
| Boolean | `bool` |
| Sequence | `list`, `tuple`, `range` |
| Mapping | `dict` |
| Set | `set`, `frozenset` |
| None | `NoneType` |

---

### 6.1 Numeric Types

**`int` (Integer)** — whole numbers, positive or negative, with no decimal point.

```python
marks = 95
temperature = -10
print(type(marks))   # <class 'int'>
```

**`float` (Floating Point)** — numbers containing a decimal point.

```python
price = 99.99
pi_value = 3.14159
print(type(price))   # <class 'float'>
```

**`complex`** — numbers with a real and imaginary part, written as `a + bj`.

```python
z = 3 + 4j
print(type(z))       # <class 'complex'>
```

> **Important — Exam Point**
>
> Python has no separate `double` type like C/C++. All decimal values in Python are represented using `float`, which internally uses double precision.

---

### 6.2 Text Type: `str`

### Definition

A **string** is a sequence of characters enclosed in single, double, or triple quotes, used to represent text.

```python
name = "Alice"
message = 'Welcome to Python'
paragraph = """This is a
multi-line string."""
```

### Common String Operations

```python
first_name = "John"
last_name = "Doe"

# Concatenation
full_name = first_name + " " + last_name
print(full_name)        # Output: John Doe

# Repetition
print("Hi! " * 3)       # Output: Hi! Hi! Hi!

# Length
print(len(full_name))   # Output: 8

# Indexing (0-based)
print(full_name[0])     # Output: J

# Slicing
print(full_name[0:4])   # Output: John
```

**Explanation:**

- `+` joins two strings together — this is called **concatenation**.
- `*` repeats a string a given number of times.
- `len()` returns the number of characters in the string.
- Indexing with `[ ]` accesses a character at a specific position, starting at `0`.
- Slicing `[start:end]` extracts a portion of the string, including `start` but excluding `end`.

> **Important — Common Mistake**
>
> Strings in Python are **immutable** — once created, individual characters cannot be changed directly (e.g., `full_name[0] = "M"` raises a `TypeError`). A new string must be created instead.

---

### 6.3 Boolean Type: `bool`

### Definition

A **boolean** represents one of two values, `True` or `False`, typically produced as the result of a logical or comparison operation.

```python
is_active = True
is_logged_in = False

print(type(is_active))   # <class 'bool'>

print(10 > 5)              # Output: True
print(10 == 5)              # Output: False
```

Internally, Python treats `True` as `1` and `False` as `0`:

```python
print(True + True)    # Output: 2
```

> **Important — Interview Tip**
>
> A classic interview question: *"What are falsy values in Python?"*
> Besides `False` itself, Python treats `0`, `0.0`, `""` (empty string), `[]`, `{}`, `()`, and `None` as **falsy** in a boolean context. Everything else is generally **truthy**.

---

### 6.4 `NoneType`

### Definition

`None` is a special constant representing the **absence of a value**.

```python
result = None
print(type(result))   # <class 'NoneType'>
```

`None` is commonly used as a placeholder before a variable is assigned a real value, or as a default return value for functions that don't explicitly return anything.

### 6.5 Introduction to Sequence and Collection Types

A short preview, since these are also data types — full detail comes in later lectures.

```python
# List - ordered, mutable collection
fruits = ["apple", "banana", "cherry"]

# Tuple - ordered, immutable collection
coordinates = (10, 20)

# Dictionary - key-value pairs
```

|---|---|---|---|

| `set` | No | Yes | No |

## 7. Type Conversion (Type Casting)

### Definition

**Type conversion** is the process of converting a value from one data type to another — either automatically by Python (**implicit**) or manually by the programmer (**explicit**).

### Implicit Conversion

Python performs this automatically when it is safe to do so, such as combining an `int` and a `float`.

```python
x = 5        # int
y = 2.0      # float
z = x + y    # Python automatically converts x to float
print(z)          # Output: 7.0
print(type(z))    # <class 'float'>
```

### Explicit Conversion

The programmer manually converts a value using built-in functions.

```python
a = "25"
b = int(a)         # string to int
print(b + 5)        # Output: 30

c = 10
d = str(c)          # int to string
print("Value: " + d)   # Output: Value: 10

e = 3.99
f = int(e)          # float to int — decimal part is truncated, not rounded
print(f)              # Output: 3

g = "3.14"
h = float(g)         # string to float
print(h)               # Output: 3.14
```

> **Important — Exam Point**
>
> `int(3.99)` returns `3`, **not** `4`. Converting a float to an int **truncates** the decimal part; it does not round. Use the built-in `round()` function if rounding is required.

### Common Conversion Functions

| Function | Purpose |
|---|---|
| `int()` | Converts to integer |
| `float()` | Converts to floating-point number |
| `str()` | Converts to string |
| `bool()` | Converts to boolean |
| `list()` | Converts to list |
| `tuple()` | Converts to tuple |

## 8. Basic Input and Output

### 8.1 Output with `print()`

```python
print("Hello")
print("Sum is:", 15)
print("Value of x is", 10, "and y is", 20)
```

Passing multiple comma-separated values to `print()` displays them separated by spaces automatically.

### Formatted Output with f-strings

An **f-string** (formatted string literal) allows variables to be embedded directly inside a string using `{ }`.

```python
name = "Alice"
age = 21

print(f"My name is {name} and I am {age} years old.")
```

> **Important — Interview Tip**
>
> f-strings (introduced in Python 3.6) are now the preferred way to format strings, replacing older methods like `%` formatting and `.format()`. Interviewers often ask you to demonstrate all three — knowing f-strings well is usually sufficient for modern Python roles.

### 8.2 Input with `input()`

### Definition

The `input()` function pauses program execution and waits for the user to type a value. The value returned is **always a string**, regardless of what the user types.

```python
name = input("Enter your name: ")
print("Hello,", name)
```

### Converting Input to Other Types

Since `input()` always returns a string, numeric input must be explicitly converted before performing arithmetic.

```python
age = input("Enter your age: ")
age = int(age)              # convert string to integer
print("Next year you will be", age + 1)

# Equivalent single-line version
age = int(input("Enter your age: "))
```

> **Important — Common Mistake**
>
> Forgetting to convert `input()` results in a `TypeError` when arithmetic is attempted, since Python cannot add an integer to a string. For example, `input("Enter age: ") + 1` fails — the string must be converted with `int()` first.
