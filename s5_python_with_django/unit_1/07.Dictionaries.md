# Dictionaries
 
### What is a Dictionary
 
A dictionary is an unordered (but insertion-ordered since Python 3.7), mutable collection of key-value pairs. Each key must be unique and immutable. Values can be of any type and can repeat.
 
A dictionary is essentially a lookup table. Given a key, it retrieves the associated value in O(1) time — constant time, regardless of how many key-value pairs exist. This is possible because Python uses a hash table internally to store dictionaries.
 
```python
# Creating a dictionary
config = {
    "model": "bert-base",
    "learning_rate": 2e-5,
    "batch_size": 16,
    "epochs": 10,
    "dropout": 0.1
}
 
# Empty dictionary
empty = {}
also_empty = dict()
 
# From keyword arguments
person = dict(name="David", age=25, role="Developer")
 
# From a list of key-value pairs
pairs = [("a", 1), ("b", 2), ("c", 3)]
d = dict(pairs)    # {"a": 1, "b": 2, "c": 3}
```
 
### How Dictionaries Work Internally
 
Python dictionaries use a hash table. When you store a key, Python computes a hash value from the key and uses that hash to determine where to store the value in memory. When you look up a key, Python computes its hash again and jumps directly to the correct location — no scanning needed. This is why lookup is O(1).
 
This also explains why keys must be immutable — the hash of an object must not change. If a list were used as a key, and the list was modified, the hash would change and the key could no longer be found. Strings, integers, floats, and tuples are hashable. Lists, sets, and dicts are not.
 
### Why Dictionaries are Central in ML
 
Dictionaries appear everywhere in ML workflows:
- Hyperparameter configurations — `{"lr": 0.001, "batch_size": 32}`
- Class label mappings — `{0: "cat", 1: "dog", 2: "bird"}`
- Word to index mappings — `{"the": 0, "cat": 1, ...}` (vocabulary)
- Training history — `{"loss": [...], "accuracy": [...]}`
- API request and response bodies — JSON is parsed into dicts
- Model configurations in HuggingFace, PyTorch, etc.
### Accessing Values
 
```python
config = {"lr": 0.001, "batch": 32, "epochs": 50}
 
# Direct access by key — raises KeyError if key does not exist
config["lr"]          # 0.001
config["missing"]     # KeyError — program crashes
 
# Safe access with get() — returns None by default, or a specified default value
config.get("lr")              # 0.001
config.get("optimizer")       # None — no error
config.get("optimizer", "adam")   # "adam" — returns default if key missing
 
# Checking if a key exists before accessing
"lr" in config          # True
"dropout" in config     # False
"lr" not in config      # False
```
 
### Modifying Dictionaries
 
```python
config = {"lr": 0.001, "epochs": 10}
 
# Adding a new key — just assign to it
config["dropout"] = 0.1
 
# Updating an existing key — same syntax
config["lr"] = 0.0001
 
# update() — add or update multiple keys at once from another dict or kwargs
config.update({"batch_size": 32, "optimizer": "adam"})
config.update(warmup_steps=500, weight_decay=0.01)    # kwargs style
 
# Merging two dicts (Python 3.9+) — the | operator
defaults = {"lr": 0.001, "epochs": 10}
overrides = {"lr": 0.0001, "dropout": 0.2}
merged = defaults | overrides    # overrides takes priority for matching keys
# {"lr": 0.0001, "epochs": 10, "dropout": 0.2}
 
# Removing keys
del config["epochs"]              # removes key — KeyError if key does not exist
config.pop("dropout")             # removes and returns the value
config.pop("missing", None)       # safe pop — returns None if key is absent
config.popitem()                  # removes and returns the last inserted pair as a tuple
config.clear()                    # empties the entire dictionary
 
# setdefault() — adds a key with a default value only if the key does not exist
config.setdefault("optimizer", "adam")    # adds "optimizer": "adam" only if key missing
config.setdefault("lr", 0.01)            # does nothing — lr already exists
```
 
### Iterating Over Dictionaries
 
```python
params = {"lr": 0.001, "batch": 32, "epochs": 50, "dropout": 0.1}
 
# Iterating over keys — this is the default behavior
for key in params:
    print(key)
 
# Iterating over values only
for value in params.values():
    print(value)
 
# Iterating over key-value pairs — most commonly used
for key, value in params.items():
    print(f"{key}: {value}")
 
# Getting views — these reflect changes to the dict in real time
params.keys()      # dict_keys(['lr', 'batch', 'epochs', 'dropout'])
params.values()    # dict_values([0.001, 32, 50, 0.1])
params.items()     # dict_items([('lr', 0.001), ...])
 
# Convert to list when you need a static snapshot
list(params.keys())
```
 
### Dictionary Comprehensions
 
Just like list comprehensions, dictionary comprehensions build a new dictionary from an iterable in a concise one-line syntax.
 
```python
# Syntax: {key_expression: value_expression for item in iterable}
 
# Basic — squaring numbers
squares = {x: x**2 for x in range(1, 6)}
# {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}
 
# With condition — filtering
scores = {"Alice": 90, "Bob": 55, "Charlie": 80, "Diana": 45}
high_scorers = {name: score for name, score in scores.items() if score >= 80}
# {"Alice": 90, "Charlie": 80}
 
# Inverting a dictionary — swap keys and values
original = {"a": 1, "b": 2, "c": 3}
inverted = {v: k for k, v in original.items()}
# {1: "a", 2: "b", 3: "c"}
 
# Word to index mapping — NLP vocabulary building
vocab = ["the", "cat", "sat", "on", "mat"]
word_to_idx = {word: idx for idx, word in enumerate(vocab)}
# {"the": 0, "cat": 1, "sat": 2, "on": 3, "mat": 4}
 
# Word length mapping
words = ["apple", "banana", "cherry"]
word_lengths = {word: len(word) for word in words}
# {"apple": 5, "banana": 6, "cherry": 6}
```
 
### Nested Dictionaries
 
A dictionary value can itself be a dictionary. This is used to represent hierarchical or structured configurations.
 
```python
# Model configuration — nested structure
model_config = {
    "architecture": "transformer",
    "layers": {
        "attention": {"heads": 8, "dim": 512},
        "feedforward": {"dim": 2048, "activation": "relu"},
        "dropout": 0.1
    },
    "training": {
        "optimizer": "adam",
        "lr": 0.0001,
        "warmup_steps": 4000
    }
}
 
# Accessing nested values — chain the keys
model_config["layers"]["attention"]["heads"]    # 8
model_config["training"]["lr"]                  # 0.0001
 
# Safe nested access using get() to avoid KeyError at any level
heads = model_config.get("layers", {}).get("attention", {}).get("heads", 4)
```
 
### Common Dictionary Patterns in ML
 
```python
# Class index to label mapping
idx_to_label = {0: "cat", 1: "dog", 2: "bird"}
label_to_idx = {"cat": 0, "dog": 1, "bird": 2}
 
# Convert predictions to label names
predictions = [0, 2, 1, 0]
labels = [idx_to_label[p] for p in predictions]
# ["cat", "bird", "dog", "cat"]
 
# Training history — collecting metrics per epoch
history = {"loss": [], "accuracy": [], "val_loss": [], "val_accuracy": []}
# In training loop:
history["loss"].append(0.45)
history["accuracy"].append(0.88)
 
# Find best epoch from history
best_epoch = history["val_accuracy"].index(max(history["val_accuracy"]))
```
