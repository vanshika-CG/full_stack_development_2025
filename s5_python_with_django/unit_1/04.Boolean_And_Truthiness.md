# Booleans and Truthiness
 
### What is a Boolean
 
A Boolean is a data type that can hold exactly one of two values: `True` or `False`. It represents the result of a logical condition — something is either true or not true.
 
In Python, `bool` is actually a subclass of `int`. This means `True` behaves like the integer `1` and `False` behaves like the integer `0` in mathematical operations. This is not just a quirk — it is by design and is used throughout Python code.
 
```python
is_trained = True
use_gpu = False
 
type(True)     # <class 'bool'>
type(False)    # <class 'bool'>
 
# Booleans are subclasses of int
int(True)      # 1
int(False)     # 0
 
# Arithmetic with booleans works
True + True    # 2
True * 5       # 5
False + 10     # 10
 
# Counting True values in a list — common ML pattern
results = [True, False, True, True, False]
sum(results)   # 3 — counts the number of True values
```
 
### Comparison Operators
 
Comparison operators compare two values and return a Boolean result.
 
```python
x = 10
y = 20
 
x == y     # False — equal to
x != y     # True  — not equal to
x > y      # False — greater than
x < y      # True  — less than
x >= y     # False — greater than or equal to
x <= y     # True  — less than or equal to
```
 
Python allows chaining comparisons in a natural mathematical way, which is not available in most other languages:
 
```python
x = 15
0 < x < 100          # True  — same as (0 < x) and (x < 100)
0.0 < 0.001 < 1.0    # True  — useful for validating learning rate
10 <= x <= 20        # True
```
 
### Logical Operators
 
Logical operators combine multiple Boolean expressions.
 
`and` returns `True` only if both sides are `True`. If the left side is `False`, Python does not even evaluate the right side — this is called short-circuit evaluation.
 
`or` returns `True` if at least one side is `True`. If the left side is `True`, Python does not evaluate the right side.
 
`not` inverts the Boolean value.
 
```python
a = True
b = False
 
a and b    # False — both must be True
a or b     # True  — at least one must be True
not a      # False — inverts True to False
not b      # True  — inverts False to True
 
# Combining conditions
epoch = 15
loss = 0.05
 
epoch > 10 and loss < 0.1    # True  — both conditions met
epoch > 20 or loss < 0.01    # False — neither condition met
not (loss > 0.5)             # True  — loss is not greater than 0.5
 
# Multiple conditions
x = 5
(x > 0) and (x < 10) and (x != 7)    # True
```
 
### Truthiness and Falsiness
 
Every Python object has an inherent Boolean interpretation. When an object is used in a condition — like in an `if` statement — Python internally calls `bool()` on it to determine if it is truthy or falsy.
 
This concept is called truthiness. It allows you to write cleaner conditions without explicitly comparing to zero or empty.
 
**Values that are Falsy — evaluate to False in a condition:**
 
- `False`
- `0` — integer zero
- `0.0` — float zero
- `0j` — complex zero
- `""` — empty string
- `[]` — empty list
- `()` — empty tuple
- `{}` — empty dict or empty set
- `None`
**Everything else is Truthy — evaluates to True in a condition.**
 
```python
# Checking truthiness explicitly
bool(0)          # False
bool(1)          # True
bool(-5)         # True  — any non-zero number is truthy
bool(0.0)        # False
bool(0.001)      # True
 
bool("")         # False
bool("hello")    # True
bool(" ")        # True  — a space character is not empty
 
bool([])         # False
bool([1, 2])     # True
bool([0])        # True  — the list has one item, even if that item is 0
 
bool({})         # False
bool({"a": 1})   # True
 
bool(None)       # False
```
 
### Using Truthiness in Conditions
 
Truthiness lets you write more concise and readable code:
 
```python
data = []
 
# Verbose way
if len(data) == 0:
    print("No data loaded")
 
# Pythonic way using truthiness
if not data:
    print("No data loaded")
 
# Checking if a variable has a value
model_name = ""
if not model_name:
    model_name = "default_model"    # assign default only if empty
 
# Checking for None
result = None
if result is None:
    print("No result yet")
 
# The 'or' trick for default values
config = None
value = config or "default_value"    # if config is falsy, use the default
 
name = "David"
display = name or "Anonymous"        # name is truthy, so display = "David"
 
count = 0
count = count or 1                   # count is 0 (falsy), so count = 1
```
 
### Short-Circuit Evaluation
 
Python evaluates `and` and `or` expressions lazily — it stops as soon as the result is determined. This has practical implications.
 
With `and`: if the left side is False, the whole expression is False regardless of the right side. Python skips the right side entirely.
 
With `or`: if the left side is True, the whole expression is True regardless of the right side. Python skips the right side.
 
```python
# Practical use — guard against calling a function on None
data = None
if data is not None and len(data) > 0:    # safe — len() not called if data is None
    print("Data has items")
 
# Unsafe version — would crash
if len(data) > 0:    # TypeError if data is None — len(None) fails
    print("Data has items")
```
 
### Identity vs Equality
 
`==` checks if two values are equal. `is` checks if two variables point to the exact same object in memory. These are different things.
 
```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a
 
a == b      # True  — same values
a is b      # False — different objects in memory
a is c      # True  — c and a point to the same object
 
# The correct and recommended way to check for None, True, False
x = None
x is None     # True  — correct
x == None     # True  — works but not the Pythonic way
 
y = True
y is True     # True  — correct for True/False/None checks
```
