# Tuples
 
### What is a Tuple
 
A tuple is an ordered, immutable, and indexed collection of elements. It looks exactly like a list but uses parentheses instead of square brackets — and crucially, it cannot be modified after creation. You cannot add, remove, or change elements in a tuple.
 
```python
point = (3, 4)
rgb = (255, 128, 0)
empty_tuple = ()
single = (42,)      # single-element tuple — the trailing comma is required
                    # without the comma: (42) is just the integer 42 in parentheses
 
# Python also allows tuple creation without parentheses
coordinates = 10, 20    # this is a tuple
type(coordinates)       # <class 'tuple'>
```
 
The single-element tuple rule is important and frequently misunderstood:
 
```python
x = (5)       # this is NOT a tuple — it is just the integer 5 in parentheses
type(x)       # <class 'int'>
 
y = (5,)      # this IS a tuple with one element
type(y)       # <class 'tuple'>
```
 
### Why Tuples Exist Alongside Lists
 
Both tuples and lists are ordered sequences. The key difference is mutability. Tuples communicate intent — when you use a tuple, you are saying "this data should not change." This makes your code safer and easier to reason about.
 
Additionally, tuples are slightly faster than lists for iteration and access. Tuples can be used as dictionary keys and added to sets because they are immutable. Lists cannot.
 
In ML, tuples are used to represent things that are fixed by design: the shape of a tensor `(224, 224, 3)`, the number of units per layer `(128, 64, 32)`, or configuration pairs.
 
### Indexing and Slicing
 
Tuples support the same indexing and slicing as lists:
 
```python
t = (10, 20, 30, 40, 50)
 
t[0]       # 10
t[-1]      # 50
t[1:4]     # (20, 30, 40)
t[::-1]    # (50, 40, 30, 20, 10)
```
 
### Tuple Methods
 
Because tuples are immutable, they only have two methods — there is nothing to add, remove, or rearrange:
 
```python
t = (1, 2, 3, 2, 4, 2)
 
t.count(2)    # 3 — how many times 2 appears
t.index(3)    # 2 — index of first occurrence of 3
t.index(2)    # 1 — index of FIRST occurrence of 2
```
 
### Tuple Unpacking
 
Tuple unpacking is one of the most useful and frequently used features in Python. It assigns the elements of a tuple to individual variables in a single line.
 
```python
# Basic unpacking — number of variables must match number of elements
point = (10, 20)
x, y = point
print(x)    # 10
print(y)    # 20
 
# Unpacking works with any iterable, not just tuples
a, b, c = [1, 2, 3]      # list unpacking
p, q, r = "abc"           # string unpacking
 
# Unpacking in a for loop
coordinates = [(1, 2), (3, 4), (5, 6)]
for x, y in coordinates:
    print(f"x={x}, y={y}")
 
# Extended unpacking with * — collect remaining elements
first, *rest = (1, 2, 3, 4, 5)
# first = 1
# rest = [2, 3, 4, 5]  — rest is always a list
 
first, *middle, last = (1, 2, 3, 4, 5)
# first = 1
# middle = [2, 3, 4]
# last = 5
 
# Swapping variables — classic Python trick using tuple unpacking
a, b = 10, 20
a, b = b, a    # swap without a temporary variable
# a = 20, b = 10
```
 
### Returning Multiple Values from a Function
 
When a Python function returns multiple values separated by commas, it actually returns a single tuple. This is one of the most common uses of tuples.
 
```python
def train_one_epoch(X, y):
    # ... training code ...
    loss = 0.345
    accuracy = 0.912
    return loss, accuracy    # Python packages these as a tuple (0.345, 0.912)
 
# Unpack the returned tuple
loss, acc = train_one_epoch(X, y)
print(f"Loss: {loss}, Accuracy: {acc}")
 
# Or keep it as a tuple
result = train_one_epoch(X, y)
print(result[0])    # loss
print(result[1])    # accuracy
 
# Common pattern — returning train and validation metrics
def evaluate(model, train_data, val_data):
    train_loss, train_acc = 0.3, 0.89
    val_loss, val_acc = 0.35, 0.86
    return (train_loss, train_acc), (val_loss, val_acc)
 
(tloss, tacc), (vloss, vacc) = evaluate(model, train_data, val_data)
```
 
### Tuples as Dictionary Keys
 
Because tuples are immutable (and therefore hashable), they can be used as dictionary keys. Lists cannot.
 
```python
# Coordinate-based lookup
location_data = {}
location_data[(28.6, 77.2)] = "Delhi"
location_data[(19.0, 72.8)] = "Mumbai"
location_data[(12.9, 77.6)] = "Bangalore"
 
# Grid cell configuration
cell_config = {(0, 0): "top-left", (0, 1): "top-right", (1, 0): "bottom-left"}
 
# Using list as key — this fails
location_data[[28.6, 77.2]] = "Delhi"    # TypeError: unhashable type: 'list'
```
 
### Named Tuples
 
A named tuple is a tuple where each field has a name. It gives you the immutability and performance of a tuple with the readability of a class, without needing to write a full class.
 
```python
from collections import namedtuple
 
# Define the structure — first arg is the type name, second is field names
Sample = namedtuple("Sample", ["features", "label", "id"])
 
# Create an instance
s = Sample(features=[1.2, 3.4], label=1, id=101)
 
# Access by name — readable
s.features     # [1.2, 3.4]
s.label        # 1
s.id           # 101
 
# Access by index — still works like a regular tuple
s[0]           # [1.2, 3.4]
s[1]           # 1
 
# Can be unpacked like a regular tuple
features, label, id = s
```
